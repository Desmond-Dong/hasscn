    Home Assistant 12 岁

    #活动回顾 - GitHub Universe '24

    如果你错过了，今年我们在 GitHub Universe 这个全球年度开发者大会上有相当高的存在感。2024 年是我们社区多年成长积累的成果，也出现了很多重要时刻：我们在旧金山举办了社区聚会、获得了奖项、在 GitHub 排名继续上升，还登上了大会第二天的主题演讲。

    在连续几年位居第二之后，我们现在已经成为 GitHub 上排名第一的开源项目，超过 21,000 名贡献者正在共同打造 Home Assistant。我们还获得了 Wonderfully Welcoming Award，成为对新贡献者而言第二活跃的项目。这些其实还只是我们在旧金山那一周经历的冰山一角，而我（Missy Quarry，你的社区与社交媒体经理）想把这些亮点分享给今年没能到场的每一个人。

    #社区齐聚一堂

    最初只是一次简单的参会请求，后来却演变成不仅参与 GitHub Universe 第二天的主题演讲 ❗❗，还获得了在 GitHub 总部举办社区聚会的场地。从那里开始，一切迅速展开：采访、专题面板、会后活动，这场为期 2 天的活动最终变成了 7 天的旧金山之行。这不仅是我第一次以开源项目成员的身份参加活动，也是我第一次在线下见到 Paulus 和 Frenck。Paulus 比我想象中还高 😅，而 Frenck 居然真的有腿！😱

    虽然 GitHub Universe 确实带来了很多关于我们的好消息（而且我一定会讲到），但如果我不先从我为 Home Assistant 协调的第一次线下聚会说起，那我就不是一个合格的社区经理了。😌

    我立刻意识到，我们需要为这次活动准备定制周边，于是我马上和我们的平面设计师 Clelia 一起开始动手。我们希望这些周边既能体现聚会所在地，又具备收藏价值，最后我们做成了明信片和邮票造型的徽章。

    限量版旧金山明信片和徽章

    当然，参加活动怎么能没有贴纸，所以我们把 Home Assistant 和 ESPHome 的标志结合 Open Home Foundation 徽章，做成了全息贴纸。✨

    活动本身规模不算大，但充满了精彩的交流与连接。Paulus 和 Frenck 花了一个多小时聊 Home Assistant 的现状，以及我们现在正在推进的工作，随后还回答了社区成员的问题。能和所有到场的人交流真的太棒了；把来自不同生活背景的人聚在一起总能让我充满能量，我已经开始期待下一次聚会了。👀

    #我们是第一！

    仅仅是能被邀请在第二天的主题演讲中发言，对 Home Assistant 来说就已经是一种荣幸；而我们在踏上这段旅程时并不知道的是，主题演讲还会宣布一条重磅消息：按活跃贡献者数量计算，我们已经是 GitHub 上排名第一的开源项目了！！🎉 不仅如此，在首次贡献者排名中，我们也上升到了第二名，而 Open Home Foundation 的合作伙伴 Zigbee2MQTT 现在也进入了前 10！🥳 这些内容都收录在 GitHub Octoverse 2024 报告 中，非常值得一读。

    GitHub 的朋友们在确保这条消息足够“惊喜”这件事上做得非常好。相信我，当他们把这件事公布给我们的时候，我们所有人都完全炸锅了。😎 我们还得知自己获得了 GitHub 的 Wonderfully Welcoming Award，这个奖项表彰的是我们如何吸引和支持新贡献者加入项目。

    #开源 - 全场焦点 ⭐️

    Paulus 和 Frenck 在周五、周日和周一都参加了主题演讲彩排，为周三做准备。于是我周一早上先去领了胸牌，那是一块可编程的黑客板，带有电子墨水屏，能显示我们的名字。这个活动在各种细节上的用心，真的让人很开心。🤩

    还有一场社区活动，GitHub 在那里稍微透露了一些他们将在第二天公布的内容。在分会场中，我也有机会分享我们在 GitHub 上直接管理社区时遇到的一些痛点，同时听听其他项目在平台上是怎么做的，这很有启发。

    Paulus、Frenck 和我在第一天主题演讲现场

    第一天，我们在 Open Source Zone 设了展位，而我自己还代表 Home Assistant 参加了一场专题面板。那场讨论叫做 Cha-ching! How to attract funding for your open source projects。我非常期待讲述我们的故事，因为 Home Assistant 的资金来自你们，也就是社区，通过 Home Assistant Cloud 订阅 来支持，而不是来自投资人。让我很惊喜的是，面板上的其他人也认同我们这种通过社区支持来打造可持续项目的方式！我还和一些正在做其他开源项目的人进行了很棒的交流，比如 Homebrew 和 Ladybird。

    我们的展位也非常成功，很多社区成员走过来告诉我们他们有多喜欢 Home Assistant，也有不少以前从未听说过我们的人前来了解。让我们感到意外的是，有些人还没有体验过我们在展位上演示的全新 Sections 仪表盘。能在线下直接得到这样的反馈真的非常有价值。当然，大家最后都拿了贴纸。😆

    我们的 [Open Source Zone](https://github.blog/open-source/10-projects-in-the-open-source-zone-at-github-universe-2024/) 展位

    第二天就是 Paulus 和 Frenck 登台的时刻，我很早就到场，只为了确保能在前排占个好位置。不仅线下现场人很多，Joost 还在我们的 Discord 服务器 的 Community Stage 上办了一场观看派对！

    他们留给现场观众的最后一句话：你准备构建什么？

    两位在台上的表现真的太棒了！🤩 看到 Paulus 解释 Home Assistant 为什么会被创建，以及它如何和 GitHub“让世界拥有 10 亿开发者”这一宏大目标相呼应，真的很精彩。Frenck 分享的不仅是我们为了成长为 GitHub 上最大的开放社区所做的事情，也谈到了社区如何共同塑造项目的方向。Kyle Daigle 也是 Home Assistant 用户，同时还是 GitHub COO，他和 Frenck、Paulus 一起上台，做了一场非常有趣的 Home Assistant 现场演示。Paulus 还讲述了他如何通过将项目交给 Open Home Foundation 来确保它拥有长远未来，以及该基金会如何致力于为智能家居带来隐私、选择和可持续性。你可以在这里观看他们的片段，不过我也很推荐把整场主题演讲都看完——GitHub 正在为新开发者做很多很棒的事情。😌

    主题演讲结束后，Paulus 和 Frenck 还接受了采访，甚至把我们多带来的一台 Home Assistant Green 送给了代表 Hack Club 出席的嘉宾演讲者。那天剩下的时间里，我们都待在展区里，继续和更多社区成员交流，享受这一切的乐趣。

    #与社区建立联系

    这真的是一场非常棒的活动。和 Paulus、Frenck 远程合作了几个月之后，终于在线下见到他们感觉特别好。最重要的是，能见到社区成员、获得反馈，并听他们讲述 Home Assistant 是如何改善他们生活的，这些都非常珍贵。感谢每一个愿意停下来和我们聊聊的人！🥰

    GitHub 是一个非常棒的东道主，在他们这么多年持续支持这个项目之后，能用这种方式回馈他们一点东西也让我很开心。在 Universe 上，他们给了我们一个平台，让我们能够解释社区对我们有多重要，也让我们能告诉大家：正是你的支持，让 Home Assistant 一直繁荣发展、不断壮大，并最终成为这个平台上排名第一的开源项目。下次你再看到我写博客时，应该就是我来宣布 Home Assistant Community Day 的时候了。😌
