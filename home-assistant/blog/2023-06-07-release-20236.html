    #连接并使用你现有的网络存储

    Do you have a network storage in your home? Like a QNAP or Synology NAS? This 发布 adds the ability to add and use that storage with Home Assistant!

    This long and highly requested feature is now available when running the Home Assistant Operating System. In the 设置 > System > Storage menu, you can now find a whole new set of controls to connect to your storage and use it to, for example, use your network share as a media source in Home Assistant.

    Even better: You can use it as a target for your backups so that when Home Assistant creates a 备份 for you, it will store it directly on your network storage.

    Multiple network storages are also supported; if you like, you can easily add more.

    #集成仪表盘与详情页

    The 集成 仪表盘 has gotten a major overhaul!

    There have been quite some usability issues with the previous 版本 of the 集成 仪表盘. For example, if you use ESPHome with many 设备, you are undoubtedly familiar with the annoying scrolling in that little 卡片. Or, big 错误 messages stretching those 卡片 out, messing up the whole page.

    This 发布 brings in a whole new 集成 仪表盘 that addresses many of these issues, aiming to provide a cleaner and more consistent experience.

    设备 based 集成, like ESPHome, are combined: no more scrolling. A failing 集成 no longer stretches the layout or makes the layout "jump" during retries. Custom 集成 and 集成 relying on the cloud are now more visibly marked.

    Selecting the cog icon on the 集成 卡片 will bring you to a brand new 集成 page, showing all information about the 集成:

    In the above example, the ESPHome 集成 page shows all 设备 此integrations has. All the information and options, previously squashed into the 集成 仪表盘, are now available here.

    These brand new 集成 pages are helpful in other places in Home Assistant. For example, the 设备 pages can now link back directly to the 集成 that provided it.

    #Python 3.11 与显著的性能提升

    This 发布 ships running on Python 3.11! In case you are wondering, what is that? Well, Python is the language the 核心 of Home Assistant is written in.

    Why it matters? It is seriously faster! 🚀

    If you run the Home Assistant Operating System or are using the Home Assistant 容器 安装 type: You won't have to do anything as we handle the 升级 to Python 3.11 for you.

    And that is not all in terms of performance. We've also upgraded internal parts, including a newer SQLite 版本 that should be faster. Oh! And, of course, @bdraco has been all over the place to optimize a lot (a lot lot).

    This 发布 can be expected to be a lot faster (or less consuming) of resources at runtime.

    #Z-Wave 更新

    There has been quite a bit of progress on Z-Wave in Home Assistant.

    Thanks to @raman325, the 集成 reached platinum level 🏆 on our quality scale. This means the 集成 now does things right according to best practices which should mean a better user experience in general.

    Have you ever turned on a Z-Wave 灯光 from the Home Assistant UI just to see the 切换 going off again after a couple of seconds, then after some more seconds seeing it going back to on? This long-standing issue (that even got the nickname flip/flop) for certain 设备, that are slow to respond to their 状态 has finally been addressed.

    For those of you that like to automate everything, you can now automate not only the Z-Wave 设备 状态 but also the 设备 配置. We added 设备 配置 parameter 实体 so that almost every Z-Wave 设备 setting is now available as an 实体.

    Not stopping there, we also added 实体 for 设备 connection statistics, which you can use to, for example, create a 通知 if a 设备 generates more than a certain amount of dropped messages. If you want to track a misbehaving 设备, you can enable these 传感器 to view the history of the connection performance.

    In the background, @AlCalzone is working tirelessly on the Z-Wave JS driver, which is the foundation of our Z-Wave 集成. Among other things, some 设备 are now faster to interview during addition, and more 设备 can be configured.

    #新实体！日期、时间、日期时间

    We welcome three new 实体 types in this 发布: date, time & datetime.

    These 实体 allow 集成 to provide date, time, or date/time input controls from the UI. They are very similar to the input datetime helper but are available for use by 集成.

    How the new time, date, and date/time 实体 will look like on your 仪表盘.

    They are so new that no 集成 are using them yet, but you can expect to see usage in (custom) 集成 of these soon!

    Thanks for creating those @raman325!

    #不向后兼容的变更

    All Home Assistant versions, of which the Home Assistant project manages the Python 版本 for you, have been upgraded to Python 3.11. This affects the following 安装 types:

    • Home Assistant Operating System
    • Home Assistant 容器
    • Home Assistant 受监管模式

    This might affect you when using custom 集成, as those must be compatible with Python 3.11.

    Furthermore, as of this 发布, Python 3.10 is deprecated, and support will be removed in Home Assistant 2023.8. This only affects you if you are running Home Assistant 核心 安装 type in your own Python virtual environment. Please 更新 your Python 版本 to Python 3.11.

    If you run Home Assistant OS, Home Assistant 容器, or Home Assistant 受监管模式, we have taken care of this for you; you will be using Python 3.11 automatically on 升级.

    sensor.ups_self_test_interval 传感器 now properly splits unit "days" from the raw data. Previously, it would show 7 days as the 传感器 状态. Now it shows 7 with a unit "Days".

    (@yuxincs - #93844) (文档)

    The counter 实体 no longer 恢复 their previous manually set 配置 (using the - now deprecated - counter.configure 服务) on Home Assistant startup. Instead, it will now use the 配置 as set in the helper 配置 in the UI (or as configured YAML, if used).

    This behavior is removed, as it caused 配置 issues, as the new 配置 was never applied again (but forever restored). This behavior was out of the ordinary for Home Assistant and thus now standardized.

    The value of the counter is still restored and available across Home Assistant restarts.

    (@frenck - #93344) (文档)

    The Command Line 集成 changes its YAML 配置 format from an old-style platform 配置 to a more modern 集成-style 配置 format.

    For example, if you have a 二元sensor configured previously as:

    # Example configuration.yaml entry
binary_sensor:
  - platform: command_line
    command: "cat /proc/sys/net/ipv4/ip_forward"

    That same 二元sensor is now configured under the command_line 集成 配置 as:

    # Example configuration.yaml entry
command_line:
  - binary_sensor:
      command: "cat /proc/sys/net/ipv4/ip_forward"

    Note: While this is currently not yet a breaking change, it did raise questions during our beta testing period. Hence we have added this as a breaking change to provide more context.
