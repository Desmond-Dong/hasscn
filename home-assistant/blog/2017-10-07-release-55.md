# 0.55: Tibber、DuckDNS、The Things Network、Owntrack

<a href='/home-assistant/integrations/#版本/0.55'><img src='/home-assistant/images/blog/2017-10-0.55/components.png' style='border: 0;box-shadow: none;'></a>

除了 [@w1ll1am23] 带来的 Wink 支持增强之外，这次发布还加入了大量新的组件与平台。`input_slider` 组件经过 [@BioSehnsucht] 的改造，现在变成了 [`input_number`][input_number docs]。[@tinloaf] 还新增了一个允许输入日期的功能：[`input_datetime`][input_datetime docs]。这两者都会帮助你改进自动化规则。

## DuckDNS

通过 [DuckDNS](http://www.duckdns.org) 为 Home Assistant 提供动态 DNS（DDNS 或 DynDNS）已经是个[老话题](/home-assistant/blog/2015/12/13/设置-encryption-using-lets-encrypt/)了。DuckDNS 也已集成进 [Hass.io](/home-assistant/addons/duckdns/)。而在 0.55 中，非 Hass.io 用户也终于可以通过组件获得类似功能。

## Recorder

数据清理能力得到了[改进][recorder docs]。通过 `purge_interval`，你可以定期清理较旧的事件和状态；配合 `purge_keep_days`，还可以指定希望保留多少天的数据。新的服务 `recorder.purge` 则允许你在需要时手动处理这项任务。

## OwnTracks

[Owntracks](http://owntracks.org/) 是一种便捷的设备追踪方式。过去我们提供的是依赖 [MQTT][device_tracker.owntracks docs] 的设备追踪平台，而现在借助 Owntracks 的一项新特性，我们也支持 HTTP 了。这个[新平台][device_tracker.owntracks_http docs] 不再需要 MQTT broker，而是直接通过 HTTP 请求向 Home Assistant 发送消息。

## Tibber

这个版本引入了一个新的传感器：[Tibber][传感器.tibber docs]。如果你是 [Tibber](https://tibber.com/) 用户，这个传感器可以提供当前电价。这样你就可以编写自动化，例如在电价高时关闭暖气，或只在电价低时给电动车充电。我们还计划进一步支持未来电价和历史用电数据展示。Tibber 目前仅在挪威和瑞典提供服务。

## The Things Network

[The Things Network](https://www.thethingsnetwork.org/)（TTN）是一个基于 [LoRaWAN](https://www.thethingsnetwork.org/wiki/LoRaWAN/Home) 的网络，专为 IoT 设备设计。通过这个[集成][thethingsnetwork docs]，只要设备连接到 TTN 网关，即使它们超出本地 WiFi 网络覆盖范围，你也依然可以观察它们的状态。

* OpenCV 图像 processing: Typo in classifier was fixed. ([@arsaboo] - [#9697]) ([image\_processing.opencv docs][image_processing.opencv docs]) (breaking change)
