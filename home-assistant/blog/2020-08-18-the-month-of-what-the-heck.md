# “What the Heck?!” 月（2020）

<a href='https://community.home-assistant.io/c/what-the-heck/52'><img src='/home-assistant/images/blog/2020-08-18-the-month-of-what-the-heck/social.png' style='border: 0;box-shadow: none;'></a>

欢迎来到 “What the heck?!” 月！

Home Assistant 走到今天已经将近 7 年了！在这段时间里，它成长为一个备受大家喜爱的庞大项目。不过，随着项目不断发展，有些地方可能并没有变成理想的样子，有些功能可能缺失，或者有些细节已经开始让你感到烦躁。这正是这个活动月要关注的内容。

我们知道，在 [GitHub][github-issues] 上报告问题门槛并不低。你需要有 GitHub 账号、按 issue 模板用 Markdown 填写内容，还要把报告写成开发者可以直接处理的形式。另外，我们的 issue tracker 主要用于跟踪真正的缺陷和 Bug，而不是一些小功能诉求或使用中的“别扭点”。

虽然这套流程在收集、跟踪和处理 Bug 方面很常见，也很合理，但它并不一定是收集“烦人小问题”或“小改进建议”的最佳方式。而这些细小改进，往往能让我们都更喜欢 Home Assistant。

今年 5 月，Ruby on Rails 项目办过一个[类似的活动月][rorwtf]，我们看得非常开心。现在，我们也准备来一次。

今天，我们已经开放了一个[社区论坛分类][forum]。这是一个更安全、门槛更低的地方，你可以在这里分享你使用 Home Assistant 时的 “What the heck?!” 时刻，更重要的是，你还可以参与讨论并为其他用户提出的话题投票。

Frenck 最近也在 Twitter 上做过类似尝试，他问了这样一个问题：

<blockquote class="twitter-tweet"><p lang="en" dir="ltr">Enlighten me 💡<br><br>The whole community would celebrate and be shedding happy tears of joy, if this tiny little thing was added to <a href="https://twitter.com/home_assistant?ref_src=twsrc%5Etfw">@home_assistant</a>...</p>&mdash; Franck Nijhof (@Frenck) <a href="https://twitter.com/Frenck/status/1285592609772244993?ref_src=twsrc%5Etfw">July 21, 2020</a>
</blockquote>

那条提问收到了非常精彩的回复！其中既有启发、也有很多我们自己也感同身受的痛点，还有不少非常、非常好的建议。而且这不只限于“很小的新功能”，例如：

* 我希望 Home Assistant 能提供一些系统级传感器，比如：自动化总数、当前小时处理过的触发器数量、集成总数、实体总数、事件触发数量。
* 我希望在首次安装时就能在前端上传备份。
* 你是否也被某些日志困扰，总要手动刷新和滚动才能看到最新内容？
* 你是否有很多做同类事情的模板实体，希望 Home Assistant 能自动处理这类场景？
* 你是否总要靠变通方案才能实现想要的自动化，并希望 Home Assistant 能提供更好的触发器或条件？

这些就是我们希望在这个月收集和分享的内容。

我们非常期待大家会带来哪些想法！

## 常见问题

* ***“我有很多想提的点！需要都放到一个帖子里吗？”***

  不用，请创建多个更小、更聚焦的话题。每个话题都可以被投票、跟进和解决。把很多问题堆在同一个帖子里，会让讨论、投票和处理都变得更困难。这个月你可以提交的建议数量没有限制。

* ***“所有反馈都会被修复或处理吗？”***

  这并不能保证。这个活动月的目标，是降低反馈门槛。Home Assistant 依然依赖贡献者来推进修复和改进。不过我们相信，这种方式收集反馈会对即将到来的 [Hacktoberfest][hacktoberfest] 有很大帮助。

* ***“我很希望新增某个集成，好让我能用我的设备。这里适合提吗？”***

  不适合。这个活动不是用于请求新的设备或服务集成。请改用 [“Feature Requests”][feature-requests] 论坛分类。

* ***“我发现了一个 Bug，而且我习惯用 GitHub。现在应该去哪里提？”***

  如果你熟悉 GitHub，请直接在 GitHub 上[提交 issue 报告][github-issues]。
