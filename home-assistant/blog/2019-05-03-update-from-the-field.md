# 一线近况更新

TL;DR:

* Nabu Casa 已成为 Let’s Encrypt 的赞助方，以支持安全家庭所需的基础设施。
* Ubiquiti 决定不再赞助 Home Assistant。我们是友好结束合作的。Paulus 已从 Ubiquiti 转为受雇于 Nabu Casa。
* 当前的开发目标是迈向 Home Assistant 1.0。
* 我们正在与制造商建立合作关系：努力让我们的集成获得认证，并让制造商更容易为自家产品贡献和维护集成。

我们已经有一段时间没有聊过我们“征服世界”的计划了，所以现在是时候从 Home Assistant 总部带来一次简短更新。

去年，Ubiquiti Networks 聘请了我，也就是 Home Assistant 的创始人 Paulus Schoutsen，全职投入到改进 Home Assistant 的工作中。这极大地帮助项目朝着 1.0 迈进。期间，Home Assistant 增加了认证系统、设备与区域的概念、用于配置集成的 UI，以及全新的 Lovelace UI，等等。

去年 9 月，在我们五周年之际，我们推出了 Nabu Casa。Nabu Casa 是一家为让 Home Assistant 变得更好而成立的公司。它通过提供 Home Assistant Cloud 服务，并将收入用于支持 Home Assistant 项目来实现这一目标。借助 Home Assistant Cloud，你只需一次选择，就可以为 Home Assistant 安装启用安全的远程访问，并集成 Google Assistant 与 Amazon Alexa。

作为对 Home Assistant 持续支持的一部分，Nabu Casa 为 Home Assistant 提供了托管社区和部分构建系统所需的基础设施。今年年初，我们聘请了 Hass.io 的创始人 Pascal Vizeli，全职投入 Home Assistant。就在这个月，我们还开始赞助 Let’s Encrypt，也就是那个为安全远程访问 Home Assistant 提供免费 SSL 安全证书的服务。

这个月也标志着 Ubiquiti 参与项目整整一年，同时也是他们参与的最后一个月，因为他们的计划发生了变化，决定结束这项合作。我们是在友好的氛围中分开的，我想感谢 Ubiquiti 提供了这样一个难得的机会，它给 Home Assistant 项目带来了巨大的推动。接下来，我将由 Nabu Casa 支付薪资，从而继续投身于 Home Assistant 的工作。

## 重点方向

在过去一年里，Home Assistant 社区一直在努力打造一个对用户友好的 Home Assistant 版本，也就是 Home Assistant 1.0。

我们的目标是在今年发布 Home Assistant 1.0。在 Home Assistant 1.0 中，安装和基础功能可以通过用户界面完成。高级选项则需要进入高级模式，或者通过 `configuration.yaml` 设置。更多细节可以查看去年的[年度状况更新](/home-assistant/blog/2018/11/16/状态-of-the-union/)。我们已经开始使用[这个项目看板](https://github.com/orgs/home-assistant/projects/1)来跟踪进展。

在全力推进 Home Assistant 1.0 的同时，我们也已经开始为 1.0 发布之后的下一项目标做准备：为制造商和集成提供更好的支持路径。

我们已经开始与合作伙伴一起为集成做认证，同时也在努力让制造商更容易改进或维护他们自己的集成。如果你是制造商，并且希望尽早参与，欢迎通过 <partner@nabucasa.com> 联系我们。我们计划很快公布更多相关信息。
