    Home Assistant 核心 2023.10! 🎉

    October! 🎃 It means we've just celebrated our birthday; we are now 10 years old! 🎂 In case you've missed our birthday YouTube stream with all the cool announcements, we've summed it all up in our 10 year Home Assistant 博客 post.

    Even more exciting news, as this is Home Assistant's year of the voice: Chapter 4 of this endeavor has been announced and will be streamed live on YouTube on 12 October 2023, at 12:00 PDT / 21:00 CEST! Be sure to tune in! I've seen some early demos... All I'm going to say is: It is mind-blowing! 🤯

    As for this 发布, because of all of the above, it is a bit smaller than usual in terms of features. I promise that chapter 4 is going to make it all worth it. Yet, it contains tons of updates, bug fixes, and a couple of beautiful new features for the tile 卡片! 🤩

    Enjoy the 发布!

    #焕然一新的漂亮 Logo！

    In case you've missed it, we have refreshed the Home Assistant logo! 🤩

    This was done for the 10th anniversary of Home Assistant, and you can read all about this iteration in this 博客 post. You will start seeing this logo in many more places in the upcoming weeks, but, it is already included this 发布!

    #全新的 My Home Assistant 按钮

    It is not just the logo that got a facelift; we have beautiful new buttons for My Home Assistant as well 🤩.


    If you're unfamiliar with My Home Assistant, it's a free online tool that enables websites, 文档, chat platforms, and blogs to link to places on your Home Assistant server without knowing its address or being exposed to the internet. This ensures your privacy while still allowing a seamless experience.

    In our 文档, you'll find those buttons to, for example, add a new 集成 to your Home Assistant with a single click! Try it yourself! This button below will open up your Home Assistant's interface and show the 集成 you have 已安装:

    So, if you see these buttons showing up somewhere, you now know what they do! You can easily create your own buttons and links to share on the create link page.

    #温控实体的全新 Tile 卡片功能

    In the last 发布, we added lots of new features for the tile 卡片, including features specifically for controlling your 温控 设备. This 发布, we are adding even more!

    @Weissnix4711 contributed a feature that allows you to add buttons for the presets of your 温控 设备 to the tile 卡片. Next, @piitaya took it a step further: You can now select which presets you want to show and if you want them in a dropdown list instead of a series of buttons.

    The screenshot above shows the same tile 卡片, but one with the buttons and one with the dropdown.

    In the options for this tile 卡片 feature, you'll be given the choice if you prefer to show icons or if you rather have a dropdown. You can also pick which presets you want to show/hide.

    #Tile 卡片的选择实体功能

    But there is more goodness for the tile 卡片. @piitaya also added the dropdown from above as a feature for the select 实体, allowing you to select an option from a list of options.

    This feature can be used for any select 实体 but also your custom dropdown helpers! (also known as input_select 实体).

    #密码管理器与 Home Assistant

    Using a password manager is arguably one of the best things you can do to improve your online security. But using them with Home Assistant, depending on which one you have been using, has been a bit of a pain.

    If you've been using one of these password managers that had issues with Home Assistant, you'll be happy to know that this 发布 fixes that! For example, 1Password now works!

    #地图实体标记选项

    The map 卡片 received new options (thanks @karwosts!) for your 实体 markers that are shown on the map.

    You can now choose if you want the 实体 name to be rendered on the map or the 实体 状态, but you could also override the name altogether. You could, for example, show 状态 values, instead of the name, for the 实体 providing gas prices, so you can see where the cheapest gas is.


    When centering the map, all 实体 on the map are taken into account. However, you can also exclude certain 实体 (for example, known outliers) that should not be considered when doing that.

    This is currently only available in the YAML 配置 of the map 卡片. See our 文档 for the map 卡片 for more information.

    #其他值得注意的变更

    There are many more improvements in this 发布; here are some of the other noteworthy changes this 发布:

    • If you are using ESPHome, you will notice much smaller Home Assistant backups when you have the latest 版本 已安装!
    • The HomeKit Bridge 集成 now supports media receivers! Thanks, @bdraco!
    • @Lash-L keeps extending the Roborock 集成. In this 发布, water and last cleaned 传感器 have been added. Sweet!
    • Withings received lots of love this 发布, thanks @joostlek! It is now easier to set up and now also works with the Home Assistant Cloud webhooks for push updates!
    • @allenporter added calendar support to the Rain Bird 集成, allowing you to see your irrigation schedule in Home Assistant! Nice!
    • The System Bridge 集成 now supports 通知! Thanks, @timmo001!
    • @alexyao2015 added a button to Life360 to 触发器 a poll to 更新 the location information. Cool!
    • The picture 实体 卡片 now has a fit mode option to control how the image fits into the 卡片. This lets you decide if the image is clipped, fitted, or stretched into the displayed 卡片. Thanks @karwosts!
    • It's now easier than ever to collect Z-Wave 日志 for troubleshooting. Just enable 调试 logging!

    #不向后兼容的变更

    The HomeKit 集成 now supports media player receivers. HomeKit expects receivers to be paired in accessory mode. Existing receivers paired in bridge mode should be re-added in accessory mode.

    (@bdraco - #100717) (文档)

    The close_comm_on_error 配置 is no longer optional, sockets will always be closed on 错误. Therefore, the close_comm_on_error is now deprecated and will be removed in Home Assistant 2024.3.

    (@janiversen - #99946) (文档)

    An MQTT 门锁 with a configured 状态 topic will initialize with 状态 unknown instead of 状态 unlocked unless the 门锁 is set to optimistic mode.

    MQTT 门锁 set to optimistic mode will still be initialized with the unlocked 状态. You should check if your 自动化 are affected.

    (@jbouwh - #100943) (文档)

    The option to set a custom scan interval in NZBGet via the options UI has been removed. The 更新 interval is now every 5 seconds.

    If you want to 恢复 the custom scan interval behavior, please see the homeassistant.update_entity 服务.

    (@joostlek - #98869) (文档)

    Some of the older 二元sensor and 传感器 from the OpenTherm gateway have been deprecated since January 2021. They have now been removed. Please use the new 二元sensor and 传感器 that have been 创建 since.

    (@joostlek - #99712) (文档)

    Some 实体 names have changed. The 集成 will migrate old 实体 to new names/IDs. You may have to manually 更新 old 实体 used in, for example, your 仪表盘.

    Invalid 实体 such as "Saturation Index" are removed if the required equipment is not configured.

    (@dieselrabbit - #92475) (文档)

    The default name given to backups 创建 with the hassio.full_backup and hassio.partial_backup 服务 are no longer the UTC date and time the 备份 was initiated. Instead, the local date and time are used for the name.

    (@jpbede - #100959) (文档)

    Any existing app you have 创建 in the Twitch Developer Portal to work with 此集成 must be updated to work with Home Assistant's UI authentication flow. 更新 the app's 配置 to include https://my.home-assistant.io/redirect/oauth in its list of "OAuth Redirect URLs"

    (@joostlek - issue#101414) (文档)

    Multiple WebSocket commands have been renamed based on this change in Z-Wave JS.

    With this 发布, you will need to 更新 your zwave-js-server instance. You must use zwave-js-server 1.32.0 or greater (schema 32).

    • If you use the Z-Wave JS add-on, you need at least 版本 0.1.91.
    • If you use the Z-Wave JS UI add-on, you need at least 版本 2.0.0.
    • If you use the Z-Wave JS UI Docker 容器, you need at least 版本 9.0.0.
    • If you run your own Docker 容器 or some other 安装 method, you will need to 更新 your zwave-js-server instance to at least 1.32.0.

    (@raman325 - #100833) (文档)

    If you are a custom 集成 developer and want to learn about breaking changes and new features available for your 集成: Be sure to follow our developer 博客. The following are the most notable for this 发布:

    • Home Assistant is participating in Hacktoberfest 2023!
    • New way of excluding 状态 属性 from recording
    • The websocket command entities/source has been 修改
