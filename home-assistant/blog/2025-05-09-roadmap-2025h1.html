    #2025 年路线图：通过集体智慧打造真正的智能家居

    自从我们为 Home Assistant 推出我们的第一个路线图 以来已经一年了，这意味着是时候推出今年的第一个更新了！到 2025 年，我们的目标是让家庭中的每个人都更容易使用 Home Assistant，并通过使智能家居更加直观、主动和用户友好来将其提升到一个新的水平。 🚀 其重点是帮助家庭助理了解其环境中的设备，并通过新项目设备数据库对与家庭助理配合使用的每个设备进行编目。我们将以家庭助理的方式做到这一点——通过我们社区的力量和集体智慧。 （……但这还不是全部！该路线图还遮盖了我们计划开展的许多产品领域，从自动化到语音到仪表盘[等等]（#and-thats-not-all-for-thisyear）！） 如果您对我们的路线图不熟悉或对它们如何指导 Home Assistant 的开发感到好奇，请务必查看 我们的 Home Assistant 路线图简介。一如既往，您的反馈非常有价值，因此请在评论中告诉我们您的想法。

    #在路上的一年

    去年，我们推出了我们的第一个路线图——这不仅仅是纸上的计划。它标志着 Home Assistant 的新篇章：我们明确了方向，设定了雄心勃勃的目标，并邀请整个社区一起参与。 在 2024 中，我们专注于构建具有高家居批准系数的智能家居：帮助智能家居的维护者和居民从 Home Assistant 中释放更多价值，而无需成为专家。这是通过改进所有家庭成员都会交互的接触点（例如自动化、仪表盘和语音交互）来实现的，同时为我们的高级用户和管理员保持平台的功能和深度。 真正的挑战在于从头开始构建路线图。这意味着不仅要仔细研究每一个功能请求并确定其优先级，还要提高透明度并在我们的组织和社区内建立对路线图的信任，将其作为可以帮助指导项目开发的灯塔。感谢全球社区的反馈、能量和贡献，我们取得了巨大进步。在过去的一年中，我们解决了一些最大的痛点，例如拖放仪表盘改进、自动化分组、标准语音助手硬件、更好的自动化备份等等。 现在，到 2025 年，我们已准备好为下一步的重大进步奠定基础：通过我们的集体努力和智慧，让家庭助理“真正智能”。

    #智能家居管理员作为发明家

    家庭认可因素意味着成功的智能家居不仅适合安装它的人，而且适合居住在那里的每个人。无论是您的伴侣、孩子还是室友，我们都希望家庭助理能够为整个家庭提供直观和支持。 在某些方面，成为智能家居管理员就像成为发明家（或者，正如我们所说的，产品经理）。要成为一名伟大的发明家，您需要了解您的用户。他们的需求是什么？您可以发明什么解决方案来解决他们的问题？你怎么知道你解决了他们的问题？您能改进该解决方案吗？ Home Assistant 为我们的管理员提供了所有构建块，让他们可以构建他们想要的任何内容并发挥他们的想象力，更好的是，通过我们去年所做的改进，它比以往任何时候都更容易！然而，这也是白板的诅咒。为利基问题发明有趣且超个性化的解决方案是家庭助理的主要吸引力，但有时，智能家居的维护可能是一件苦差事——并不是每个问题都需要重新发明轮子。 今天的现实是，根据我们的社区调查，只有 46% 的长期 Home Assistant 用户的合作伙伴和 27% 的孩子直接与 Home Assistant 互动。这是一个很大的差距，它凸显了一个更广泛的挑战：即使是最有经验的智能家居管理员也并不总能全面了解家庭的需求。在出现问题之前，他们可能不会意识到什么会有帮助。期望每个人都成为从自动化到仪表盘再到新用户培训的专家是不合理的。

    #从共享智慧到集体智慧

    现在，如果智能家居管理员想要改进他们的设置，他们必须去寻找——通过我们的社区论坛、GitHub 存储库、Reddit 线程、YouTube 视频、Discord 聊天……凡是你能想到的——试图找到适用于他们特定情况的提示和技巧。你们每天都会互相分享令人难以置信的智慧，但其中大部分都是分散的、短暂的，或者如果没有大量的定制就很难应用。 我们相信现在是时候采用更智能的系统了。 如果家庭助理可以从我们社区的最佳想法中学习并利用这些知识主动提出改进建议，结果会怎样？如果它可以根据其他人在类似情况下所做的事情来指导用户怎么办？这就是我们正在努力实现的愿景：利用集体智慧为真正的智能家居提供动力。

    为此，家庭助理需要“了解”家里有什么。每个家庭内部都有人、空间和设备。今年，我们将首先帮助 Home Assistant 了解设备的以下方面：(1) 上下文：每个设备是什么以及如何使用它，以及 (2) 信息：与 Home Assistant 配合使用的每个设备及其支持方式。

    #将设备置于上下文中

    如今，Home Assistant 中的大多数设备都以一堆实体的形式出现：这里有一个温度传感器，那里有一个开关。这是设计使然——我们希望为您的系统配置提供最大的灵活性，并将继续这样做。然而，它也并非没有缺点。

    例如，我们并不总是知道这些实体在一起形成了一个冰箱，它本质上是一个自己的迷你房子：它可能有温度传感器、门传感器、灯光等等，但家庭助理不能将它们假设为房间内的任何其他传感器或灯光。如果没有这种背景，家庭助理除了让用户从头开始构建自己的仪表盘和自动化之外，无法做太多事情。 当你添加更多的设备时，实体就会堆积起来，家庭助理开始失去家里实际情况的线索。随着您添加更多设备，您的智能家居应该变得更加强大，但代价是维护它们的难度也成倍增加。 当我们将设备放在上下文中时，一切都会发生变化。冰箱不再是一系列实体——它变成了一个实际的设备，可以拥有专用的仪表盘、默认的自动化和上下文语音命令。

    以下是上下文如何在整个平台上解锁更简化的体验：

    • 语音： 如果 Assist 知道设备的区域，它当前可以获取正确的传感器。未来，我们还可以智能地从语音查询中排除不相关的传感器。例如，“厨房的温度是多少？”不应返回冰箱的内部温度。
    • 仪表盘：集成可以从一开始就提供定制的配比和仪表盘，提供更加简化的体验。例如，我们可以提供冰箱仪表盘。它也适用于其他设备。想象一下汽车仪表盘或 3D 打印机仪表盘。
    • 自动化：通过了解用户拥有的设备以及它们的常用方式，我们可以智能地建议我们的社区创建自动化。例如，可以提供“冰箱门打开”或“冰箱漏水”警报，而无需从头开始手动构建它们。 这将是一个核心架构变化，将在未来几个月内提出和讨论。我们正在以我们的贡献者在集成中已经分享的内容为基础：有关设备如何构造以及它们应该如何表现的知识。我希望这将为我们的贡献者提供一个框架，以令人兴奋的新方式为我们的代码贡献仪表盘和自动化蓝图。

    #设备数据库简介

    为了使这一切顺利进行，我们还需要一个集中的、结构化的地方来存储和共享设备知识。这就是设备数据库——开放家庭基金会的一个全新项目——发挥作用的地方。 将其视为由社区创建、策划和验证的事实来源。我们在这里收集一切信息，从元数据（如电量使用或红外代码）到事实信息（设置说明，如如何添加或重置设备），再到现实世界的设置见解和社区创建（如自动化示例）。它将包含用户有意提交到数据库的信息，未经您的明确同意，不会自动收集任何内容。 它不仅仅是将所有设备文档集中在一处。我们希望将所有关于设备的集体经验推广到一个地方，这比在线网站和聊天室更容易访问、更集中、更结构化。 借助设备数据库，新老用户都可以轻松做出明智的选择，并根据实际使用体验挑选出最适合他们的家庭助理兼容设备，使他们能够根据开放的家居价值观（隐私、选择和可持续性）轻松做出决策。 这些数据将在 Home Assistant 和其他开放家庭基金会项目中使用和交叉引用，并作为其集成的支柱。例如，假设未来的红外集成可以从数据库中访问现成的红外代码中受益。

    这不是我们可以单独建造的东西。我们依靠全球社区的力量将这一愿景变为现实，并通过我们的基础设施、工程工作和合作伙伴关系来支持这一愿景：

    • 开放家庭基金会 提供框架来帮助我们轻松、安全地收集知识并进行协作。
    • 我们正在与我们的商业合作伙伴 Nabu Casa 合作，打造最可靠的硬件天线，支持更多开放协议（例如 Z-Wave、Zigbee、蓝牙等），以扩展 Home Assistant 生态系统。
    • 与家庭助理合作计划 正在得到加强，以便更轻松地为该平台找到值得信赖的高品质设备，正如您在过去几个月中看到的众多新合作伙伴 的加入。 结果呢？在这个系统中，管理员无需独自解决所有问题。 Home Assistant 可以通过汲取社区的集体智慧来建议、指导、调整和引导用户走向正确的方向（而不是像大型科技公司那样不由自主地抓取用户数据）。

    #这还不是今年的全部！

    虽然设备背景和设备数据库是今年路线图的主要主题，但这并不意味着我们要放弃其他不适合的内容。我们正在继续致力于产品体验的其他部分：

    • 对我们的自动化操作和条件进行彻底改造，使它们更易于使用且更强大。
    • 导航和设计系统大修，以改进功能发现。
    • 持续改进，使仪表盘更易于开箱即用，包括新的默认仪表盘等。
    • 增强了对用户、访客和公共访问的隐私控制（是的，我知道这一点。）
    • 更轻松地设置音乐助手。
    • 使 Assist 更具对话性，例如确认和澄清查询的能力。
    • 继续探索使用法学硕士来改善语音以外的整体用户体验。 与此同时，我们正在努力使我们的路线图更加公开，以便您可以与我们讨论和跟踪我们的进展！敬请关注。

    #为真正的智能家居奠定基础

    该路线图与浮华的最新科技时尚（现在是什么？）或抽象功能无关。这是为了创造持久的基础，让我们的社区能够构建比我们所有人共同创造的更伟大的东西：一个能够学习、成长和适应的智能家居平台——充分尊重隐私、选择和可持续性。 该路线图的中心主题是，让家庭助理变得更智能，首先要了解环境，例如了解设备是什么以及它们的使用方式，我们将利用这种集体智慧来增强用户体验的主要支柱——自动化、语音和仪表盘。 我们正在使智能家居的维护变得更加容易，让它不再像一项强制性的苦差事，同时仍然保持修补的乐趣（比如，你想在 YAML 上疯狂），并通过创建工具来帮助管理员解决他们甚至不知道的问题。我们相信，随着我们的高级用户不断修补他们的系统，他们的创造和发现也将造福和提升每个用户的智能家居。 我们迫不及待地想在今年剩下的时间里致力于这些举措！让我们一起构建它！ 🚀👩🏻‍🚀👩🏼‍🚀👨🏼‍🚀 - [玛德琳娜]、[JLo] 和 [劳拉] [Madelena]: https://github.com/Madelena [JLo]: https://github.com/jlpouffier [Laura]: https://github.com/laupalombi
