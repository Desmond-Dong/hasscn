    #备份系统的另一次迭代

    在我们的一月发布中， 我们对备份系统进行了第一次检修。在二月发布中， 我们根据您的反馈进行了迭代。在本次发布中，我们致力于更多 是时候进一步迭代和改进备份了。 让我们深入了解吧！

    #按位置设置备份保留策略

    此版本添加了一项备受期待的功能：设置备份保留的能力 每个地点的政策。如果您有多个备份，这尤其有用 在 Home Assistant 中配置的位置。

    您的备份位置之一的存储空间可能比 另一个，或者您希望在一个位置保留比另一个位置更多的备份。 例如，也许你的 Google Drive 只想保留最近 3 个备份，而本地 NAS 服务器则想保留 25 个备份。

    #升级 Home Assistant 操作系统前先备份

    更新 Home Assistant 操作系统就像按下一个按钮一样简单。虽然升级流程本身已经带有失败时的回退机制，但在开始升级前，最好仍然在多个位置保留一份最近的备份。 为了帮助你做到这一点，Home Assistant 操作系统的更新对话框现在加入了一个新选项，可在开始升级前先创建备份。选择该选项后，会按照自动备份的设置创建一份备份；如果你尚未配置自动备份，则只会备份设置、历史记录和附加组件。随后，它会在升级开始前上传到所有已启用的备份位置。

    这样，您可以确保之前已备份过系统 升级，即使出现超出安全保障范围的问题 操作系统升级。

    #设置升级时的默认备份偏好

    上述升级前备份的选项很棒，但是关于 在混合升级之前切换以创建备份。你们中有些人喜欢总是 升级前备份，而其他人则依赖计划备份而不是 每当升级零件时都希望被备份过程所困扰 的家庭助理。 为了让这两个组都满意，我们在备份设置页面添加了一个新设置 设置升级切换之前备份的默认行为。

    当您将选项设置为“升级前备份”时，将启用切换 默认情况下，否则将默认为禁用。还有一个额外的 选项也可以为附加组件设置此行为，因此您可以根据自己的需要调整它 喜欢。

    #在重启时等待备份完成

    另一个虽小但重要的改进是：如果你在备份进行过程中重启 Home Assistant，系统会等待备份完成后再执行重启。

    这可以避免备份进行中出现意外，也能防止你因为重启 Home Assistant 而中止备份过程。

    #为 Home Assistant Cloud 用户提供大量新的文本转语音语音变体

    #Z-Wave 长距离支持和改进的 Smart Start

    本次发布为 Z-Wave 集成和整体使用体验带来了不少增强功能。我们的商业合作伙伴 Nabu Casa 正在开发一个几乎不算秘密的 Z-Wave 天线项目，而我们希望 Home Assistant 也已准备就绪，能够提供尽可能好的用户体验。 本次发布，我们改进了设置新 Z-Wave 设备的整体体验，包括通过扫描设置新的 Z-Wave 设备的改进支持智能启动二维码。 Z-Wave 智能启动 QR 扫描现在可以在我们的 移动配套应用程序 中本地运行，消除浏览器限制。添加的设备立即可见 家庭助理，即使设备尚未开机。一旦通电或重新启动， 设备将自动添加到您的 Z-Wave 网络。本次发布最令人兴奋的部分是添加了 Z-Wave 远程支持。这是一项新的 Z-Wave 技术，允许设备进行通信比传统的Z-Wave设备传输距离更长。这尤其是如果您有需要放置在远处的特定设备，则很有用， 例如，门上的接触式传感器或车道上的邮箱。

    您可以选择是否将拓扑 Z-Wave 设备添加到现有网络状网络或直接使用远程连接。

    扫描具有远程功能的设备的二维码后，您将获得 可以选择将其添加到现有网状网络或直接连接 使用长距离。两者各有利弊，推荐使用网状网络 大多数设备，但如果您有一个设备需要放置在较远的地方， 远程可能是更好的选择。
