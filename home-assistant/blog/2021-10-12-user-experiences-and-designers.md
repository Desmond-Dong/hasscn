# 用户体验与设计师

<img src='/home-assistant/images/blog/2021-10-ux/social.png' style='border: 0;box-shadow: none;'>

大约两个月前，我的手机一直被通知震个不停。在宣布我将作为第一位全职 UX 设计师加入 Home Assistant 后，我收到了许多非常暖心的推文。这份热情同样体现在超过 500 人报名参加我们的 [user research group](https://forms.gle/5NSabxHJ3qkp7Vke9)，以及我在 [Twitter 上分享](https://twitter.com/matthiasdebaat) 的设计想法下收到的大量评论中。

## 用户体验

Home Assistant 拥有一个庞大的用户群体，而他们有一个共同目标：实现以本地控制和隐私优先为核心的家庭自动化。根据我们的 [analytics](https://analytics.home-assistant.io)，目前有超过 85,000 名用户选择匿名分享他们的数据，但我们知道实际用户远不止这些。每个家庭和每位用户都不尽相同，他们的需求甚至可能彼此冲突。而这正是 UX 的价值所在。正如 [Scott Jenson](https://youtu.be/Mjup7Mrj7uU) 在他精彩的主题演讲中所说，UX 提供的是一种视角。用户是谁？他们的任务是什么？他们的痛点是什么？他们想实现什么？

在接下来的一段时间里，我会研究智能家居用户都有哪些类型、他们的需求是什么，以及有哪些障碍阻碍他们达成目标。

## 设计师

仅在去年一年，就有超过 8,000 名 GitHub 贡献者参与了 Home Assistant。这真的非常了不起！但设计师与开发者的比例只有 1:8,000，这很容易让我们陷入设计的后台流程中，而忽略了真正的用户。这可能导致界面不一致，或者本可以更易用的设计没有做到位。目前我们正专注于三个方向。

### UX 贡献者

我们希望让设计师参与贡献，就像开发者参与一样容易。设计师能贡献的地方很多，例如让灯光分组更易使用，或者让家庭自动化的入门过程更顺畅。如果你有一个非常希望我们看看的设计方案？如果你想帮助我们构建自己的设计系统？如果你有用户测试经验？如果你有战略层面的设计想法？想组织线上设计工作坊？或者只是想认识其他设计师？欢迎来到 [devs\_ux Discord channel](/home-assistant/join-chat)，或加入 [GitHub discussions](https://github.com/home-assistant/前端/discussions/10147)。

### 工具链

我们希望为 Home Assistant 创建和迭代设计稿这件事足够简单，让任何感兴趣的人都能直接参与。为此，我们一直在努力把构成 Home Assistant 界面的组件重新打造为一个 [DesignKit for Figma](https://www.figma.com/community/file/967153512097289521/Home-Assistant-DesignKit)。这让任何人都可以试验 Home Assistant 界面，并快速尝试不同概念。欢迎亲自试试，也欢迎分享你的设计。

### 文档

清晰的指南有助于兼顾速度与质量。这也是为什么我们正在编写设计文档，并会很快发布。它的目标是帮助设计师和开发者做出更好的决策。如果你有建议，或者想帮助我们完善这些指南，欢迎来 [devs\_ux Discord channel](/home-assistant/join-chat)。
