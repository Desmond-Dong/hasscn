    #2024.9：Sections 变大了

    Home Assistant 2024.9！🎉

    许多北半球的社区成员利用好天气休息了一会儿。一段时间以来，Home Assistant的版本持续增长，这一版本的规模可能不像以前的版本那么庞大，但它仍然包含一些非常酷的功能和公告。

    我们希望您本月也能在智能家居内外享受一段美好时光。

    #Sections 变大了

    为了给用户带来更多仪表盘定制和组织选项，我们在 Sections 视图中加入了一些新能力，这也真正释放了可调整卡片尺寸的潜力。这些强大的自定义功能可能会影响卡片或分区布局的可预测性，尤其是在桌面与移动视图之间切换时，因此我们发现它更适合用在较大的卡片上。和往常一样，怎么使用，完全取决于你。

    #更宽的部分

    部分可以扩展以跨越多个部分。例如，现在您可以让安全网络头源跨越两个部分或拥有一个巨大的三部分宽地图。部分和部分将继续遵循 Z 网格规则，如果新的较宽部分不适合屏幕的特定部分，这可能会导致空白。如果您不喜欢各部分之间的空白，您可以选择_密集_布局。密集部分布局选项将尝试遵循 Z 网格规则，但会尽可能填充空白。这可能会影响各部分的顺序和可预测性，但会最大化屏幕空间。

    当切换到较小的屏幕时，这些较大的部分将恢复为一个部分宽（如果这会导致某些大的反馈问题，我们建议使用我们的设备可见性设置）。

    #全宽卡片

    更大的分区也意味着可以容纳更大的卡片。不只是单纯拉伸布局，现在卡片还有一个新选项，可以让它始终占满整个分区宽度。

    #徽章

    谈到仪表盘，我们在上次发布中介绍 的新样式徽章现在更加可配置。

    您可以自己决定要显示的元素，而不是使用显示模式。例如，这允许您隐藏徽章的图标，并仅显示您想要在徽章中显示的设备的状态。

    #追踪你未追踪的能量

    四月份，我们在能源仪表盘中引入了单个设备的新能源图表。这使您可以将所有能源监控聚合到一个图表中，从而使您可以快速查看家里最大的耗能设备。非常感谢 @karwosts，该图的原始创建者，现已更新。

    我们不可能对每个设备都进行能源跟踪，如果您有整个家庭的能源跟踪（例如智能电表或 DIY 的东西），更新的图表将计算差异并显示您家的未跟踪使用情况。因此，跳上你的能量仪表盘并开始解决你的幻象能量负载。

    #LLM 代理选项增加

    当我们在语音助手中添加使用大型语言模型(LLM)的功能时，开始的对话代理数量有限。其中包括基于云的OpenAI和Google AI集成，以及本地运行的Ollama集成。现在，这三个人都可以控制您家中的设备。

    @Shulyaka 现在通过 Anthropic 添加了另一个云选项 - 再次感谢您为我们在 Home Assistant 中尝试 AI 提供了另一个选项。设置它可能与其他会话代理对接不同，因此请查看文档。我们已经更新了我们的 LLM 排行榜 以包括 Anthropic。 Claude 3 Haiku 和 Claude 3 Sonnet 在我们的基准测试中得分都非常高，现在名列前茅。 根据基准测试的结果，我们将更便宜的 Claude 3 Haiku 设置为 Anthropic 集成的默认模型。

    @allenporter添加了配置Ollama集成的上下文窗口大小的功能。默认大小已增加，性能提高了20%！

    #Aqara 加入 与 Home Assistant 配合使用

    我们最近宣布 有四款 Aqara 设备加入我们的与 Home Assistant 配合使用 计划。真正令人兴奋的是，这些都是 Matter 设备，Aqara 将成为第一个使用我们的“Works via Matter with Home Assistant”徽章的合作伙伴，提供了一些与 Home Assistant 令人增强的 Matter 集成一起使用的优秀选项。

    更重要的是，这将是自我们最近更新该计划以来第一个加入的合作伙伴，这意味着我们自己测试了逐个设备认证。这使您可以轻松了解哪些内容为您带来Home Assistant可以带来的最佳体验。

    #开源合规

    我们投入了大量的工作来确保 Home Assistant 从上到下都是真正的开源。这意味着确保我们的集成所依赖的每个库都使用合规的许可证。我们现在有超过 2,800 个集成，所以你可以想象，这不是一个小任务。我们与代码所有者和库开发人员合作，煞费苦心地检查每个库现在都使用 OSI 批准的许可证。这可能看起来像一个小程序性的事情，但对于保持集成和 Home Assistant 的功能在遥远的未来至关重要。如果您不知道，库是许多设备和服务集成所依赖的功能，调用通过 工艺品 位于 Home Assistant 项目之外的附加协议代码特定。如果个人或公司无法再维护库，适当的开源许可证允许其他人拾取并继续使用。

    #集成

    感谢我们的社区跟进新的举动 对以及现有的改进！你们都很棒🥰

    #新集成

    我们欢迎本版本中的以下新集成：

    • [人择对话]，由 @Shulyaka
      添加 使用 Anthropic 的 AI，例如 Claude 3.5 Sonnet 作为 Home Assistant 中的对话代理。
    • Nice G.O.，由 @IceBotYT
      添加 与 Nice 兼容的车库门和开门器集成。
    • [富士通 FGLair]，由 @crevetor
      添加 使用 Ayla IoT 平台的富士通 HVAC 设备（通过 FGLair 应用程序控制）。
    • SMLIGHT，由 @tl-sl
      添加 从 Home Assistant 管理 SMLIGHT SLZB-06 适配器。
    • Roth Touchline SL，由 @jnsgruk
      添加 Roth Touchline SL 地暖系统的新集成。
    • [耶鲁]，由 @bdraco
      添加 Yale Home 品牌现在拥有自己的集成，将不再使用 August 集成。如果您将 Yale Home 设备与 August 集成一起使用，则应迁移到新的 Yale 集成。

    此版本还有一个新的虚拟集成。虚拟集成 是由其他（现有）集成处理的存根，以帮助 可发现性。这些是新的：

    • ArtSound，由 LinkPlay 提供，由 @dukeofphilberg 制作

    #对现有集成的值得注意的改进

    不只是新增了集成，现有集成也在持续增强。以下是一些值得注意的改进：

    • Mosquitto MQTT Broker 附加组件现在可以在你设置 MQTT 时自动安装并启动。干得漂亮，@jbouwh！
    • @allenporter 为 Nest 集成添加了事件实体。
    • @bdraco 可以更轻松地添加电池供电的 Shelly 设备，您不再需要手动设置出站 WebSocket 配置。
    • MotionBlinds 现在有一个按钮可用于您最喜欢的位置。谢谢@starkillerOG。
    • @joostlek 添加了从 Home Assistant 更新您的 AirGradient 设备的功能。
    • @CM000n 在 APsystems 集成中添加了诊断二元传感器。
    • Mastodon 集成现在显示有关您账户的信息，例如您有多少关注者。谢谢@andrew-codechimp！

    #现在可以通过 UI 设置

    虽然大多数可以直接从Home Assistant进行设置 用户界面，有些只能使用 YAML 配置使用。我们继续前进 对 UI 进行更多集成，使每个人都更容易使用它们 设置和使用。

    你现在可以通过 UI 添加数字型 [模板助手]。感谢 @dougiteixeira！

    #告别以下内容

    以下内容也已不再可用：

    • Asterisk Call Detail Records 和 Asterisk Voicemail。它们此前 已弃用并现已删除。他们一直在使用已弃用的邮箱功能 不再可用。

    #向后不兼容的更改

    我们尽力避免对现有功能进行更改，这可能会导致 意外地影响您的 Home Assistant 安装。不幸的是，有时， 这是不可避免的。

    我们始终确保记录这些更改，以使过渡变得简单 对你来说是可能的。此版本具有以下向后不兼容的更改：仅当您将记录器集成与 MySQL 或使用时，此通知才适用 PostgreSQL 数据库。如果您使用默认的 SQLite 数据库， 你可以忽略它。

    上次发布包括数据库升级/迁移，增加了 我们可以为标识符字段存储的数字。

    尽管标记为成功，但并非所有迁移都成功。这个 发布将再次进行迁移，但会捕获不成功的迁移。

    这可能会导致数据库升级比平时花费更长的时间，具体取决于 您的数据库的大小。

    请耐心等待升级过程完成。请勿中断 升级过程，因为这可能会导致数据库损坏。
