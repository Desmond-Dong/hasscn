# 介绍 Works with Home Assistant 计划

:::note
**这篇博客已过时。** 有关合作伙伴计划的最新信息，请访问我们的 [Works with Home Assistant 网页](https://works-with.home-assistant.io/)。
:::

在 Home Assistant 中，我们集成了超过 1000 个不同的 API。其中大多数集成由 Home Assistant 社区创建并维护。多年来，已有不少公司主动与我们的社区合作，提供样品和工程支持。在少数情况下，我们也看到一些公司接手维护其产品在 Home Assistant 中的集成。

遗憾的是，曾有几次公司在最初做出贡献后便不再回应，导致用户疑惑为什么新设备没有得到支持。我们希望保护用户，避免他们为家庭购买那些无法与 Home Assistant 良好协作的产品。

今天，我们推出了 Works with Home Assistant 计划，让制造商能够展示他们对 Home Assistant 及其社区的支持与承诺。

该计划要求制造商维护其产品在 Home Assistant 中的集成，提供良好的用户体验，提供产品样品，并向我们提供工程联系人以便升级处理问题。作为回报，制造商将能够在其产品和文档中使用“Works with Home Assistant”徽章。Works with Home Assistant 计划的条款将通过 Nabu Casa 与制造商双方签署的协议来执行。

<br><br>

借助 Home Assistant，我们始终在帮助用户了解，在购买新产品时应优先考虑本地控制和开放标准。这一点也体现在 “Works with Home Assistant” 徽章中。

有些制造商正在开发通过 Z-Wave、Zigbee 或 Matter（即将支持）等标准接入 Home Assistant 的产品。在这些情况下，集成由 Home Assistant 社区和 Nabu Casa 维护。这些公司仍然可以成为 Works with Home Assistant 计划的成员，但无需承担集成维护工作。

获得 [Made with ESPHome 计划](https://esphome.io/guides/made_for_esphome.html)批准的产品，将有资格作为其 Made with ESPHome 会员权益的一部分，使用 “Works locally with Home Assistant” 徽章。

制造商如需了解更多信息，请参阅我们的 [Works with Home Assistant 门户](https://partner.home-assistant.io)。
