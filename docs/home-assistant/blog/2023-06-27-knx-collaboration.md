我们很高兴宣布，[KNX Association](https://www.knx.org/knx-en/for-professionals/index.php) 已正式认可 [Home Assistant 中的 KNX 集成](/home-assistant/integrations/knx/)。接下来，KNX Association 将与 Home Assistant 的 KNX 团队协作，帮助维护 KNX 集成，并进一步优化用户体验。

KNX 是一种可本地运行的家庭与楼宇自动化标准，广泛用于商业和住宅场景。它主要但并不完全依赖一条集中式数据总线，这类线路通常会在房屋建造时一并铺设。KNX 也提供了可通过无线频段或 IP 基础设施运行的扩展方案，并与数百家智能家居厂商及数千名安装商开展合作。

Home Assistant 中的 KNX 集成让用户可以借助 Home Assistant 的能力，把 KNX 设备与其他智能家居设备统一到一起，并享受美观的界面、强大的自动化能力以及本地语音控制。用户还可以配置 Home Assistant，将非 KNX 设备反向暴露给 KNX 网络。

<img src='/home-assistant/images/blog/2023-06-27-knx-collaboration/panel.png' alt="The KNX 面板 in Home Assistant" class='no-shadow'>
<br><br>

Home Assistant 在 2016 年 7 月的 [Home Assistant 0.24](/home-assistant/blog/2016/07/16/sqlalchemy-knx-join-simplisafe/) 中首次引入了 KNX 集成。自 7 年前发布以来，社区一直在持续推动它演进。目前该集成主要由 [marvin-w](https://github.com/marvin-w) 和 [farmio](https://github.com/farmio) 负责开发，他们已将其打磨到白金质量等级。

像 KNX 这样的组织开始优先推动与 Home Assistant 的无缝集成，并直接与集成开发者合作，这确实让人振奋。我们也非常高兴看到事情正朝着这样的方向发展。
