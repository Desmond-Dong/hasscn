今天是发布日！我们很高兴发布 Home Assistant 0.83。你可能已经注意到，我们采用了新的发布时间表：从周五改为周三（这次晚了一天）。整体仍然是和以前一样的双周发布节奏，只是整体向后移动了 5 天。

那这次有什么新内容？很多。先说一个贡献者里程碑：这次发布中，后端有 99 位贡献者。也就是说，自上次发布以来的 2.5 周里，有 99 个人提交了如今已成为 Home Assistant 一部分的代码。前端还有另外 15 位贡献者。看到社区能够一起协作，打造出顶级的家庭自动化软件，真的很了不起。

我要特别感谢 [@MartinHjelmare]，他在审阅 PR 和尽可能提供帮助方面做得非常出色。干得漂亮！

好了，来看几个重点。首先，感谢 [@pbalogh77]，我们现在支持 Fibaro Home Center 网关了。如果你之前已经在 Fibaro 的 Z-Wave 网络上投入很多，现在就可以无缝集成到 Home Assistant，并与其他集成一起使用。

第二个亮点是 Google Assistant 现在支持门锁。默认可执行上锁；如果你希望支持解锁，需要前往 Home Assistant Cloud 设置中启用解锁，或者如果你使用的是手动配置，就在配置中添加 `allow_unlock: true`。

<p class='img'>
  <img src='/home-assistant/images/blog/2018-11-release-83/google-unlock.png' alt='Screenshot of the Google Assistant 卡片 on the cloud 面板.'>
  查看哪些实体已暴露给 Google，并启用门锁解锁功能。
</p>

前端团队也没闲着。扩展发布说明见[这里](https://github.com/home-assistant/home-assistant-polymer/pull/2077)，但最大的亮点是新的购物清单卡片。[@iantrich] 把购物清单面板浓缩成了 Lovelace 卡片的形式。

另外，如果你喜欢 GPS 追踪：这个版本让 [OwnTracks][OwnTracks docs] 更容易上手了。如果你更偏好尽可能全面的数据，[@ludeeus] 还新增了 Traccar 支持，并配套了由 [@frenck] 提供的 [Hass.io add-on](https://community.home-assistant.io/t/community-hass-io-add-on-traccar/81407?u=frenck)。

请注意，这次发布包含一项迁移，会添加索引以加速日志簿查询。首次启动可能需要一点时间，请耐心等待。

## 发布 0.83.3 - December 3

我们发现了一个问题：如果某些用户仍在使用旧版认证进行 API 访问，但 UI 访问并未使用旧版认证，那么这些旧版 API 用户可能会变成没有任何权限的旧版认证用户。如果你遇到这个问题，我们建议你升级脚本，改用长期有效访问令牌，它能提供完全相同的功能。另一种做法是按照[这些说明](https://github.com/home-assistant/home-assistant/pull/18922)手动将旧版 API 用户加入管理员组。
