终于来了，这是 Home Assistant Android Companion 应用最常见的功能请求之一：你现在可以配置多个 Home Assistant 服务器。应用可以把传感器更新发送到所有服务器，也能轻松混搭不同服务器的部件、小组件、磁贴和控件 🎉

另外，Home Assistant 应用的预览版现已可通过 Windows 11 的 Amazon Appstore 安装，让你能快速把电脑接入 Home Assistant。

此外还有一些改动，进一步打磨了应用体验，并改进了对最新 Home Assistant 版本特性的支持。下面一起来看看这次更新！

## 配置多个 Home Assistant 服务器

有时，一个 Home Assistant 实例并不够。也许你太喜欢 Home Assistant，于是开始帮别人上手；也许你有第二个住处；或者你经常去一个也在用 Home Assistant 的地方。

在这次更新之前，如果你想访问多个服务器会比较麻烦。你可能需要切换应用、建立书签、使用不同配置文件，更别说要把 Android 设备同时连到多个服务器以使用传感器和通知了。

现在这些都成了过去式。在 **[Settings](https://my.home-assistant.io/redirect/config/)** > **Companion app** 下新增了添加第二个服务器的选项，再来一个、再来一个也行……所有服务器都会显示在应用里，你也可以继续使用熟悉的功能，比如设备传感器和推送通知。每个服务器都有独立设置，所以如果你不想在所有地方共享步数或开启位置追踪，也完全没问题。

![Screenshots of a phone showing different dashboards in the app, one of them showing a pop-up titled 'Choose server'](/home-assistant/images/blog/2023-03-30-android/multiserver.png)

这还不是全部。你还可以在应用 _外部_ 使用所有服务器，在快捷方式、通知、小组件、磁贴和控件里自由混搭。把三套房子的灯并排切换？没问题！

注意：Wear OS 应用暂不支持多服务器。

## 现已支持 Windows 11（预览）

Home Assistant Android Companion 应用现已可[通过 Amazon Appstore][via the Amazon Appstore]安装在[受支持的 Windows 11 设备][supported Windows 11 devices]上。虽然社区已经有很多不错的电脑集成方案，但 Companion 应用可以让你更容易开始，并直接受益于 Android 应用持续迭代带来的新功能。

![Screenshot of the Home Assistant Android app on a Windows 11 device](/home-assistant/images/blog/2023-03-30-android/windows.png)

Windows 有时无法向应用提供正确信息，也可能在几分钟后终止后台任务（包括传感器），因此我们将 Windows 11 支持以预览形式发布。Windows 11 对 Android 的支持仍在持续改进，我们也会在未来继续调整，提升整体体验。

[via the Amazon Appstore]: amzn://apps/android?asin=B09RP3DM5T
[supported Windows 11 devices]: https://support.microsoft.com/windows/f8d0abb5-44ad-47d8-b9fb-ad6b1459ff6c

## 持续打磨 Wear OS

Home Assistant 应用登陆 Wear OS 手表已经超过一年，而且还在持续进化。今年早些时候，我们加入了 Assist 的初步支持；在这次更新中，[@dshokouhi] 进一步通过表盘组件让访问更方便。现在只要轻点一下就能用 Assist！

实体太多，想在手表上减少干扰？以前你可以添加收藏项，它们会显示在应用顶部，但下方仍会加载全部实体。现在新增了“仅显示收藏”选项，可以减少加载内容，也能避免误操作。快试试这个更聚焦的体验，告诉我们你是否喜欢！

![Screenshots of a Wear OS device, showing an Assist complication a watchface, updated launch screen, and favorites only home screen](/home-assistant/images/blog/2023-03-30-android/wear.png)

[@SkechyWolf] 和 [@dshokouhi] 还一起改进了应用的视觉和交互，让它与现代 Wear 应用风格更一致。

[@SkechyWolf]: https://github.com/SkechyWolf
[@dshokouhi]: https://github.com/dshokouhi

## 其他值得关注的改动

这次更新还有更多内容，以下是一些亮点：

- 任何带位置信息的传感器，现在都可在 Android Auto 导航中显示。干得漂亮，[@dshokouhi]！
- [@jpelgrom] 增加了对 2023.3 新 Thread integration 的初步支持。感谢！
- [@Adi146] 和 [@dshokouhi] 改进了 Beacon Monitor 在后台和开关切换时的稳定性，体验更可靠了。
- 做了很久的自定义仪表盘，结果在手机上因为字体太小或自动旋转而错位？现在不会了，[@dmiales] 调整了设置来避免这类问题！
- 有些通知你只想在手表上看？上个版本已加入向 Wear OS 应用发送通知支持，这次 [@dshokouhi] 继续推进，新增了 TTS 通知支持。

这次更新还有更多人修复了各种 bug，比如 [@Suxsem] 和 [@JBassett]。感谢所有贡献者。如果你有改进应用的好想法，或者发现了可以修复的问题，欢迎[加入论坛讨论][join us on the forums]或[在 GitHub 贡献代码][contribute on GitHub]。
