## 备份系统的另一次迭代
在我们的[一月发布](/home-assistant/blog/2025/01/03/release-20251/#overhaul-of-the-备份系统)中，
我们对备份系统进行了第一次检修。在[二月发布](/home-assistant/blog/2025/02/05/release-20252/#iterating-on-backups)中，
我们根据您的反馈进行了迭代。在本次发布中，我们致力于更多
是时候进一步迭代和改进备份了。
让我们深入了解吧！
### 按位置设置备份保留策略
此版本添加了一项备受期待的功能：设置备份保留的能力
每个地点的政策。如果您有多个备份，这尤其有用
在 Home Assistant 中配置的位置。<img class="no-shadow" alt="显示为 Microsoft OneDrive 集成设置的备份保留策略的屏幕截图。" src="/home-assistant/images/blog/2025-05/backups-retention-per-location.png" />

您的备份位置之一的存储空间可能比
另一个，或者您希望在一个位置保留比另一个位置更多的备份。
例如，也许你的 Google Drive 只想保留最近 3 个备份，而本地 NAS 服务器则想保留 25 个备份。

### 升级 Home Assistant 操作系统前先备份

更新 Home Assistant 操作系统就像按下一个按钮一样简单。虽然升级流程本身已经带有失败时的回退机制，但在开始升级前，最好仍然在多个位置保留一份最近的备份。
为了帮助你做到这一点，Home Assistant 操作系统的更新对话框现在加入了一个新选项，可在开始升级前先创建备份。选择该选项后，会按照自动备份的设置创建一份备份；如果你尚未配置自动备份，则只会备份设置、历史记录和附加组件。随后，它会在升级开始前上传到所有已启用的备份位置。
<img class="no-shadow" alt="显示 Home Assistant 操作系统更新对话框的屏幕截图，其中有一个新的切换，会询问您是否要在升级之前备份。" src="/home-assistant/images/blog/2025-05/backup-before-upgrade-home-assistant-operating-system.png" />

这样，您可以确保之前已备份过系统
升级，即使出现超出安全保障范围的问题
操作系统升级。
### 设置升级时的默认备份偏好
上述升级前备份的选项很棒，但是关于
在混合升级之前切换以创建备份。你们中有些人喜欢总是
升级前备份，而其他人则依赖计划备份而不是
每当升级零件时都希望被备份过程所困扰
的家庭助理。
为了让这两个组都满意，我们在备份设置页面添加了一个新设置
设置升级切换之前备份的默认行为。
<img class="no-shadow" alt="显示备份设置中升级之前的备份切换首选项的屏幕截图。" src="/home-assistant/images/blog/2025-05/backups-select-preference-backup-before-update.png" />

当您将选项设置为“升级前备份”时，将启用切换
默认情况下，否则将默认为禁用。还有一个额外的
选项也可以为附加组件设置此行为，因此您可以根据自己的需要调整它
喜欢。
### 在重启时等待备份完成

另一个虽小但重要的改进是：如果你在备份进行过程中重启 Home Assistant，系统会等待备份完成后再执行重启。

<img class="no-shadow" alt="显示 Home Assistant 在重新启动之前等待备份的屏幕截图。" src="/home-assistant/images/blog/2025-05/backups-prevents-restart-when-in-progress.png" />

这可以避免备份进行中出现意外，也能防止你因为重启 Home Assistant 而中止备份过程。

## 挑选实体
我们正在努力改善用户界面中显示的内容的上下文。这是为了
确保您始终知道您正在查看哪个设备或实体，无论怎样
您在 Home Assistant 界面中的位置。在上一篇文章中，我们已经对此进行了一个值得注意的小更改。
我们在实体信息对话框中添加了设备和区域，因此您始终
确切地知道您正在查看哪个实体。
在本次发布中，我们改进了 UI 实体选择器中提供的上下文。这是当您在其中选择实体时看到的下拉菜单，例如，动作、自动化、
或脚本。该选择器现在还将显示设备和区域名称！
<img class="no-shadow" alt="屏幕截图显示了新的实体选择器，提供了更多上下文并改进了搜索功能。" src="/home-assistant/images/blog/2025-05/improved-entity-picker.png" />

此增强功能逐渐消除了您手动重命名的需要
实体或设备包含位置信息——为您提供更好的体验
只需更少的工作即可获得开箱即用的体验。
不仅改进了选择器的视觉外观，而且改进了搜索
功能也已彻底修改！ 🔎 寻找确切的实体
由于更智能的搜索，您正在寻找的内容现在变得更加容易
算法并在匹配中添加更多上下文信息
过程。
:::tip
**你可能会注意到实体ID不再显示在选择器中！**
我们认为我们正处于一个你可以完全依赖 UI 和
不再需要实体ID了。但是，如果您**确实希望**看到
实体 ID 在此下拉列表中，**您可以在您的个人资料设置中启用它**。
<img class="no-shadow" alt="屏幕截图显示了改进的实体选择器，在我的用户个人资料中启用了在实体选择器中显示实体 ID 选项。" src="/home-assistant/images/blog/2025-05/improved-entity-picker-with-entity-ids.png" />

为此，请在左下角选择您的个人资料图片
Home Assistant UI，并在 **[用户设置](https://my.home-assistant.io/redirect/profile/)** 下启用**在选取器中显示实体 ID** 选项。
:::

## Z-Wave 长距离支持和改进的 Smart Start

本次发布为 Z-Wave 集成和整体使用体验带来了不少增强功能。我们的商业合作伙伴 [Nabu Casa](https://nabucasa.com) 正在开发一个几乎不算秘密的 Z-Wave 天线项目，而我们希望 Home Assistant 也已准备就绪，能够提供尽可能好的用户体验。
本次发布，我们改进了设置新 Z-Wave 设备的整体体验，包括通过扫描设置新的 Z-Wave 设备的改进支持智能启动二维码。
Z-Wave 智能启动 QR 扫描现在可以在我们的 [移动配套应用程序](https://companion.home-assistant.io/) 中本地运行，消除浏览器限制。添加的设备立即可见
家庭助理，即使设备尚未开机。一旦通电或重新启动，
设备将自动添加到您的 Z-Wave 网络。本次发布最令人兴奋的部分是添加了 Z-Wave 远程支持。这是一项新的 Z-Wave 技术，允许设备进行通信比传统的Z-Wave设备传输距离更长。这尤其是如果您有需要放置在远处的特定设备，则很有用，
例如，门上的接触式传感器或车道上的邮箱。
<p class='img'>
<img alt="显示添加具有长距离支持的 Z-Wave 设备的屏幕截图。" src="/home-assistant/images/blog/2025-05/zwave-long-range.png" />
您可以选择是否将拓扑 Z-Wave 设备添加到现有网络状网络或直接使用远程连接。</p>

扫描具有远程功能的设备的二维码后，您将获得
可以选择将其添加到现有网状网络或直接连接
使用长距离。两者各有利弊，推荐使用网状网络
大多数设备，但如果您有一个设备需要放置在较远的地方，
远程可能是更好的选择。

## 改进了在自动化和脚本中使用模板时的 UI 体验
如果您是高级用户，在自动化和脚本中使用模板，您
可能熟悉 UI 回退到使用 YAML 的事实任何包含模板的动作块。
[@karwosts] 来救援！他已经研究了一个解决方案来改善这一点
经验。
因此，对于整个块来说，现在只需要使用纯 YAML 即可回退到包含动作模板的字段的代码编辑器。这个
意味着 UI 的其余部分（对于所有其他动作字段）保持不变，并且您
仍然可以使用 UI 来编辑其余部分。
正如您在下面的屏幕截图中看到的，亮度是使用模板设置的，
这没问题，因为其余的动作仍然可以在 UI 中编辑。
<img class="no-shadow" alt="屏幕截图显示了在使用模板时所做的自动化改进，在这些情况下保持 UI 的其余部分可用。" src="/home-assistant/images/blog/2025-05/template-automation-improvement.png" />

这对于高级用户来说是一个很大的改进，因为它允许您使用 UI
对于大多数自动化或脚本，同时仍然能够使用模板
在需要的地方。它还使理解自动化或脚本变得更容易，
您可以在 UI 中看到模板的上下文。
那里有很大的进步[@karwosts]！ 🙌

## 发现 Home Assistant 正在发现什么

Home Assistant 会持续扫描你的网络，以通过各种发现协议查找新设备和集成。这会极大提升你的
体验，因为它可以让您轻松地向您的家庭添加新设备
助理安装。
然而，有时，您可能想知道家庭助理正在看到什么
或者想知道为什么某个设备没有被发现。为了帮助解决这个问题
[@bdraco]添加了 DHCP、mDNS/ZeroConf 和 UPnP/SSDP 的浏览工具让您可以看到家庭助理所看到的内容。这是很有技术含量的，但是
这是查看网络中正在发生的情况的好方法。
<img class="no-shadow" alt="新的ZeroConf/mDNS浏览器的屏幕截图，显示了Home Assistant的内部发现过程。" src="/home-assistant/images/blog/2025-05/discovery-browsers.png" />

你可以在 **设置** > **系统** > **网络** 中找到这些新工具。
