# “Supervisor 249：解决中心、稳定性和私有容器注册中心”

_Supervisor 附带我们的家庭助理操作系统，可控制您设备上的所有硬件和软件。通过用户界面带来管理、升级和维护 Home Assistant 实例的轻松体验。它可以让你专注于真正重要的事情；自动化你的家._

主管通常没有足够的更改，我们可以用每个发布来填充博客文章，它也没有固定的发布时间表。新的 Supervisor 更新会在需要时发布，但这并不意味着生态系统的这一部分没有发生任何事情！一个月前，我们发布了【主管入党】【主管入党】的帖子，回顾了过去一年的亮点。现在是版本 249 的时候了，在我们讲话时向所有人推出，它充满了新的好东西，享受吧！

## 解决中心

我们正在推出一个新的解决中心。此新功能将用于识别问题并提供解决安装问题的上下文和建议。我们对此有一个宏伟的计划！

此版本的新功能是，如果您的设备上的可用空间降至 1 GB 以下，自动更新将暂停，并且您将收到通知。这应该会给您时间来清理它。

为了帮助您释放一些空间，我们在[文档中添加了新页面][more-info_free-space]来帮助指导您完成此过程。

如果您运行 Home Assistant 受监管模式，如果主机系统没有运行所有 [required 服务][required_services]，系统可能会被标记为不受支持。很难找到系统被标记为不受支持的原因。它在启动期间写入日志，但您在 UI 中找到的日志不会回溯那么远。为了使其更易于访问，您的系统不受支持的消息现在将允许您打开一个新对话框。

![unsupported dialog](/home-assistant/images/blog/2020-10-21-supervisor-249/unsupported_dialog.png)

该对话框将列出系统被标记为不受支持的所有原因。单击该对话框中的链接将转到描述该问题的文档页面，并且您可以使用一个解决方案来使您的安装符合 [ADR-0014] 和 [ADR-0015]。

＃＃ 稳定

稳定性是新 Supervisor 版本始终关注的焦点。 Supervisor 是 Home Assistant OS/受监管模式 安装的核心部分。几乎每个发布都有某种形式的稳定性改进，这也不例外。

如果 Home Assistant 无法启动，我们现在将重建容器。这将确保在容器内的某些内容已损坏时启动新的新容器。我们还在新添加的[观察者插件][supervisor-joins-the-party-observer]中添加了此功能。

如果您的系统可用空间不足，所有自动更新将自动暂停。仍然允许手动更新，但现在我们不会自动填充该空间，这在极少数情况下可能会导致启动问题。

在添加插件存储库时，我们还添加了一个新的检查，以确保它实际上是一个插件存储库。这应该有助于避免混淆。

## 登陆页面

我们更新了您在安装 Home Assistant 时看到的登录页面。我们在其中添加了日志，以便您可以在等待时了解正在发生的事情。

还有更多！我们的移动应用程序现在能够发现登陆页面。这意味着您可以在初始化时发现新的安装，并且已经开始在移动应用程序中加入。

![landingpage](/home-assistant/images/blog/2020-10-21-supervisor-249/landingpage.png)

## 私人容器登记处

感谢 [@skateman][skateman]，您现在可以在 Supervisor 中使用私有（受密码保护）容器注册表。这允许您使用插件和其他容器，这些容器需要您将日志记录到容器注册表中才能安装它。

![private registry](/home-assistant/images/blog/2020-10-21-supervisor-249/private_registry.png)

您可以通过单击右上角的三个点在 Supervisor 面板的商店选项卡中找到它。仅当启用高级模式时此选项才可用。

## 更严格的网络管理员检查

在此版本中添加了更多检查，以确保网络管理器不仅在系统上启用，而且还管理至少一个网络接口。如果您正在运行我们的 Home Assistant 操作系统，则这不适用于您。

这种更严格的检查使我们能够为主管和插件开发者提供稳定的方式来获取主机的 IP 地址。

如果网络管理器不管理系统上的至少一个网络接口，系统将被标记为不受支持。我们在我们的文档中添加了一个[新页面][unsupported_network_manager]，它将帮助您修复它。
