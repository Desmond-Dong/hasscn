## 改进所有“选择器”
选择器是您用于选择实体设备、区域等的下拉菜单。选择器在 Home Assistant 中*提示中*，用于选择自动化、脚本和配置选项。上次发布，我们[引入了一个新的实体选择器](/home-assistant/blog/2025/05/07/release-20255/#picking-entities)，它非常受欢迎！此发布通过根据您的反馈改进搜索体验来进一步改进。
更好的是，同样改进的搜索体验现在也适用于区域、类别、楼层、标签、用户和设备选择器！无论您选择什么，它们都是一致、快速且易于使用的。
那么设备选择器呢？它现在拥有全新的外观，配有制造商徽标和受实体选择器启发的造型。

## 了解蓝牙

Home Assistant 拥有强大的[蓝牙功能]。借助像 [ESPHome 蓝牙代理] 这样的方案，你可以让蓝牙网络覆盖整个家庭。

本次发布对蓝牙集成进行了一些改进，帮助你更好地理解自己的蓝牙网络。蓝牙集成现在提供了可视化视图，显示蓝牙设备是如何连接到 Home Assistant 实例的，无论它们是直接连接到系统，还是通过蓝牙代理连接。

<img class="no-shadow" alt="屏幕截图展示了新的蓝牙连接图；甚至蓝牙代理也显示在地图中。" src="/home-assistant/images/blog/2025-06/bluetooth-connection-graph.png" />

它不仅显示当前已连接的设备，还会显示在范围内、但 Home Assistant 尚未识别的设备。这是查看周边有哪些设备，以及它们可能通过哪里连接的好方法。

另外，既然蓝牙集成有了全新的可视化界面，原本就具备类似功能的 [Zigbee 集成] 也同步更新了视觉样式。这让整个 Home Assistant 的可视化体验更加一致。

## 实验区仪表盘的迭代
在四月份的发布中，我们推出了[新的实验性区域仪表盘](/home-assistant/blog/2025/04/02/release-20254/#a-new-experimental-areas-仪表盘)，它会根据您在家中设置的区域自动生成即用型仪表盘。它使用部分和平铺调整来实现现代、干净和直观的外观——只需单击几下即可为您构建。
由于这是实验性的，因此正在不断迭代。根据您的反馈，此版本带来了一些不错的更改，以巧妙地改进体验。
我们添加了一个新的“动作”部分，其中包括脚本、自动化和场景。我们还在“其他”部分添加了数字实体（和数字助手）、按钮实体（和按钮助手）、计数器和计时器助手。这使得在您所在区域的上下文中查找和管理您的自动化和场景变得更加容易。
最后，我们将“娱乐”部分重命名为“媒体播放器”。这更明确地表明，此部分专门针对您的播放器，可能不包括您的 DIY 街机👾。

## 弃用安装方法和 32 位架构
本发布引入了重要的弃用公告。虽然它们只影响一小部分用户，但让受影响的用户了解其影响非常重要。我们写了一篇[详细博客](/home-assistant/blog/2025/05/22/deprecating-core-and-supervised-installation-methods-and-32-bit-systems/)，解释背后的原因、时间表，以及如何迁移。如果您认为自己可能受到影响，我强烈建议您阅读。

- **安装方法**：**Core** 和 **Supervised** 安装方法现已正式弃用。这些属于高级部署方式，只被少数用户使用，因为它们涉及在 Python 环境中运行 Home Assistant，或在您自己的操作系统上安装 Supervisor。今后，我们将把支持重点放在更精简、也更易维护的安装方法上：**Home Assistant OS** 和 **Home Assistant Container**。

- **32 位架构**：对旧版 32 位 CPU 架构（**i386**、**armhf** 和 **armv7**）的支持也已弃用。这些架构已经越来越少见，也给兼容性与性能维护带来了额外挑战。

为了帮助您平稳过渡，如果您的系统受到这些弃用的影响，Home Assistant 会在升级后创建一个**修复问题**。这条通知会显示在修复仪表盘中，并提供弃用说明以及如何迁移到受支持设置的指导。

需要注意的是，虽然这些安装方法和架构已经被弃用，但它们在接下来的六个月内仍会继续获得支持，直到 Home Assistant 2025.12 发布。之后您依然可以继续使用它们，但我们强烈建议您迁移。六个月支持期结束后，它们将变为不受支持状态，也就是说将不再获得更新或官方协助。

如果您没有收到这条修复通知，那您就不受影响。不过，如果您想再次确认，也可以查看[这份指南](/home-assistant/blog/2025/05/22/deprecating-core-and-supervised-installation-methods-and-32-bit-systems/#check-if-youre-affected)。我们也提供了关于[“弃用”和“不受支持”具体意味着什么](/home-assistant/blog/2025/05/22/deprecating-core-and-supervised-installation-methods-and-32-bit-systems/#what-does-deprecated-and-unsupported-mean)的说明，以及[如何迁移](/home-assistant/blog/2025/05/22/deprecating-core-and-supervised-installation-methods-and-32-bit-systems/#how-to-migrate)的详细信息。

## 侧边栏改进
如果您不知道，您实际上可以调整 Home Assistant 侧边栏的内容。您可以通过按住侧边栏顶部的“Home Assistant”文本来更改顺序并显示 /hide 项目，或者转到您的用户个人资料页面并选择“更改顺序并隐藏侧边栏的项目”。此版本对此功能带来了一些急需的改进。当您自定义侧边栏时，它现在会弹出一个漂亮而干净的对话框，允许您拖放项目以更改其顺序，或通过单击项目旁边的眼睛图标来隐藏它们。
<img class="no-shadow" alt="新对话框的屏幕截图，可用于排序或隐藏侧边栏中的项目。" src="/home-assistant/images/blog/2025-06/sidebar-customization.png" />

它最大的改进还不止这些！过去，侧边栏自定义有时只会应用在你当前使用的设备上；更麻烦的是，它偶尔还会丢失设置，或者在家庭网络与内部 URL 之间切换时，让[移动伴侣应用]中的配置“失忆”。这次，这个烦人的问题终于解决了！侧边栏自定义现在会存储在你的用户配置中，因此你的个性化布局会在所有使用 Home Assistant 的设备之间同步。

对你来说是可能的。此版本具有以下向后不兼容的更改：
Met Office [已弃用此集成使用的 Datapoint API](https://www.metoffice.gov.uk/服务/data/datapoint/datapoint-retirement-faqs)，导致其停止工作。为了解决此问题，它现已迁移到 [DataHub API](https://www.metoffice.gov.uk/服务/data/met-office-weather-datahub)。要继续使用此集成，您需要提供新的 API 密钥，可以通过注册 [DataHub](https://datahub.metoffice.gov.uk/) 并订阅 [全球现货数据集](https://datahub.metoffice.gov.uk/pricing/site-specific) 来获取该密钥。免费订阅每天提供 360 个调用，这足以让此集成发挥作用。由于新的数据源，一些传感器发生了变化：
- 天气实体提供的每小时预报现在有实际的每小时条目，而不是旧版本中的每小时 3 小时
- 集成现在公开一个具有精确可见距离（以米为单位）的 `visibility` 传感器，而不是显示范围和定性描述（例如“1-4 公里”和“差”）的 `visibility` 和 `visibility_distance` 传感器
- 集成不再暴露“每日”和“3小时”传感器；现在只有一组
- 集成传感器不再公开 `Site ID`、`Site name` 和 `Sensor ID` 属性，因为这些属性不提供任何附加值
