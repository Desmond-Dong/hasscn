# 谢谢你们

一年前，[Home Assistant 0.10][0.10] 发布。上周末我们发布了 0.35。社区一年内完成 25 次发布是非常了不起的成就，而且每一次发布都让我们大步向前。仅这一年，主仓库就收到了 2800 个 pull request，平均每天超过 7 个！

我所在公司（[AppFolio]）的创始人 Jon Walker 教过我一件事：和竞争对手相比，你能为自己创造的最大优势，就是更频繁地发布。每次发布都能让新功能更快到达用户和开发者手中。人们越早开始使用，你就越快拿到反馈，也就越快完成演进。

这就是为什么我把 Home Assistant 设定为两周一个发布周期。这样既能确保功能快速上线，也能迫使我们不要积压文档和测试工作。每两周我们都能重新出发。这也让新成员更容易参与贡献，因为发布时间清晰，大家不会担心错过某个发布窗口。

不过，两周一个发布周期也意味着社区每两周都要集中冲刺，确保一切准备就绪。很多人都参与其中，保证每个环节都到位。对你们所有人，我想说：谢谢！感谢你们投入的时间和精力，让 Home Assistant 成为最优秀的家庭自动化软件之一。

我们也要特别感谢 Python 语言的开发者，以及 Home Assistant 所依赖的所有开源库和工具的维护者。做出高质量软件绝非易事，你们都值得为自己骄傲。

同样感谢那些为开源项目免费提供服务的公司。没有这些支持，我们不可能以现在的速度和规模运转。谢谢 [GitHub]、[TravisCI]、[CloudFlare] 和 [Discourse]！

最后，感谢这个乐于助人又超棒的社区 🙇。

我们将进行一次应得的短暂休整，并在 2017 年带着更多精彩内容回归。节日快乐！

– Paulus
