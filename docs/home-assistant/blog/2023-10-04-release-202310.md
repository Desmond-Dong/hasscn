## 温控实体的全新 Tile 卡片功能

In the last 发布, we added [lots of new features for the tile 卡片](/home-assistant/blog/2023/09/06/release-20239/#lots-of-new-tile-features),
including features specifically for controlling your 温控 设备. This 发布, we are adding even more!

[@Weissnix4711] contributed a feature that allows you to add buttons for the presets of your 温控 设备 to the tile 卡片. Next, [@piitaya] took it a step further: You can now select which presets you want to show and if you want them in a dropdown list instead of a series of buttons.

<img class='no-shadow' src='/home-assistant/images/blog/2023-10/tile-card-climate-presets.png' alt='Screenshot showing the new preset feature of the tile 卡片 in both button and dropdown variants.'>

The screenshot above shows the same tile 卡片, but one with the buttons and one with the dropdown.

In the options for this tile 卡片 feature, you'll be given the choice if you
prefer to show icons or if you rather have a dropdown. You can also pick
which presets you want to show/hide.

[@piitaya]: https://github.com/piitaya
[@Weissnix4711]: https://github.com/Weissnix4711

## Tile 卡片的选择实体功能

But there is more goodness for the tile 卡片. [@piitaya] also added the dropdown from above as a feature for the select 实体, allowing you to select an option from a list of options.

<img class='no-shadow' src='/home-assistant/images/blog/2023-10/tile-card-select.png' alt='Screenshot showing the select feature for the tile 卡片 in the Home Assistant interface.'>

## 密码管理器与 Home Assistant

Using a password manager is arguably one of the best things you can do to
improve your online security. But using them with Home Assistant, depending on
which one you have been using, has been a bit of a pain.

If you've been using one of these password managers that had issues with
Home Assistant, you'll be happy to know that this 发布 fixes that!
For example, 1Password now works!

## 地图实体标记选项

The [map 卡片](/home-assistant/dashboards/map/) received new options (thanks [@karwosts]!)
for your 实体 markers that are shown on the map.

You can now choose if you want the 实体 name to be rendered on the map
or the 实体 状态, but you could also override the name altogether. You could,
for example, show 状态 values, instead of the name, for the 实体 providing
gas prices, so you can see where the cheapest gas is.

<center>
<img class='no-shadow' src='/home-assistant/images/blog/2023-10/map-features-prices.png' alt='Screenshot of a map 卡片 showing gas prices on the map.'>
</center><br />

When centering the map, all 实体 on the map are taken into account.
However, you can also exclude certain 实体 (for example,
known outliers) that should not be considered when doing that.

This is currently only available in the YAML 配置 of the map 卡片.
See our [文档 for the map 卡片](/home-assistant/dashboards/map/#label_mode) for
more information.

[@karwosts]: https://github.com/karwosts

## 其他值得注意的变更

There are many more improvements in this 发布; here are some of the other
noteworthy changes this 发布:

- If you are using [ESPHome], you will notice much smaller Home Assistant
  backups when you have [the latest 版本] 已安装!
- The [HomeKit Bridge] 集成 now supports media receivers! Thanks, [@bdraco]!
- [@Lash-L] keeps extending the [Roborock] 集成.
  In this 发布, water and last cleaned 传感器 have been added. Sweet!
- [Withings] received lots of love this 发布, thanks [@joostlek]! It is now
  easier to set up and now also works with the [Home Assistant Cloud] webhooks
  for push updates!
- [@allenporter] added calendar support to the [Rain Bird](/home-assistant/integrations/rainbird)
  集成, allowing you to see your irrigation schedule in Home Assistant!
  Nice!
- The [System Bridge] 集成 now supports 通知! Thanks, [@timmo001]!
- [@alexyao2015] added a button to [Life360] to 触发器 a poll to 更新 the
  location information. Cool!
- The [picture 实体 卡片] now has a fit mode option to control how the image
  fits into the 卡片. This lets you decide if the image is clipped, fitted,
  or stretched into the displayed 卡片. Thanks [@karwosts]!
- It's now easier than ever to collect [Z-Wave] 日志 for troubleshooting.
  Just [enable 调试 logging](/home-assistant/integrations/zwave_js#how-do-i-access-the-Z-Wave-日志)!

## 不向后兼容的变更

The HomeKit 集成 now supports media player receivers. HomeKit expects
receivers to be paired in accessory mode. Existing receivers paired in bridge
mode should be re-added in [accessory mode](/home-assistant/integrations/homekit/#accessory-mode).

([@bdraco] - [#100717]) ([文档](/home-assistant/integrations/homekit))

[@bdraco]: https://github.com/bdraco
[#100717]: https://github.com/home-assistant/核心/pull/100717

The `close_comm_on_error` 配置 is no longer optional, sockets will
always be closed on 错误. Therefore, the `close_comm_on_error` is now
deprecated and will be removed in Home Assistant 2024.3.

([@janiversen] - [#99946]) ([文档](/home-assistant/integrations/modbus))

[@janiversen]: https://github.com/janiversen
[#99946]: https://github.com/home-assistant/核心/pull/99946

An MQTT 门锁 with a configured 状态 topic will initialize with 状态
`unknown` instead of 状态 `unlocked` unless the 门锁 is set to optimistic mode.

MQTT 门锁 set to optimistic mode will still be initialized with
the `unlocked` 状态. You should check if your 自动化 are affected.

([@jbouwh] - [#100943]) ([文档](/home-assistant/integrations/MQTT))

[@jbouwh]: https://github.com/jbouwh
[#100943]: https://github.com/home-assistant/核心/pull/100943

The option to set a custom scan interval in NZBGet via the options UI has been
removed. The 更新 interval is now every 5 seconds.

If you want to 恢复 the custom scan interval behavior,
please see [the `homeassistant.update_entity` 服务](/home-assistant/common-tasks/general/#defining-a-custom-polling-interval).

([@joostlek] - [#98869]) ([文档](/home-assistant/integrations/nzbget))

[@joostlek]: https://github.com/joostlek
[#98869]: https://github.com/home-assistant/核心/pull/98869

Some of the older 二元sensor and 传感器 from the OpenTherm gateway have
been deprecated since January 2021. They have now been removed. Please use
the new 二元sensor and 传感器 that have been 创建 since.

([@joostlek] - [#99712]) ([文档](/home-assistant/integrations/opentherm_gw))

[@joostlek]: https://github.com/joostlek
[#99712]: https://github.com/home-assistant/核心/pull/99712

Some 实体 names have changed. The 集成 will migrate old 实体
to new names/IDs. You may have to manually 更新 old 实体 used
in, for example, your 仪表盘.

Invalid 实体 such as "Saturation Index" are removed
if the required equipment is not configured.

([@dieselrabbit] - [#92475]) ([文档](/home-assistant/integrations/screenlogic))

[@dieselrabbit]: https://github.com/dieselrabbit
[#92475]: https://github.com/home-assistant/核心/pull/92475

The default name given to backups 创建 with the `hassio.full_backup`
and `hassio.partial_backup` 服务 are no longer the UTC date and time the
备份 was initiated. Instead, the local date and time are used for the name.

([@jpbede] - [#100959]) ([文档](/home-assistant/integrations/hassio))

[@jpbede]: https://github.com/jpbede
[#100959]: https://github.com/home-assistant/核心/pull/100959

Any existing app you have 创建 in the [Twitch Developer Portal](https://dev.twitch.tv/console/apps/) to work with 此集成 must be updated to work with Home Assistant's UI authentication flow. 更新 the app's 配置 to include `https://my.home-assistant.io/redirect/oauth` in its list of "OAuth Redirect URLs"

([@joostlek] - [issue#101414]) ([文档](/home-assistant/integrations/twitch))

[@joostlek]: https://github.com/joostlek
[issue#101414]: https://github.com/home-assistant/核心/issues/101414

Multiple WebSocket commands have been renamed based on [this change in Z-Wave JS](https://zwave-js.github.io/node-zwave-js/#/getting-started/migrating-to-v12?id=renamed-network-heal-to-rebuild-routes).

With this 发布, you will need to 更新 your zwave-js-server instance.
You must use zwave-js-server `1.32.0` or greater (schema `32`).

  - If you use the Z-Wave JS add-on, you need at least 版本 `0.1.91`.
  - If you use the Z-Wave JS UI add-on, you need at least 版本 `2.0.0`.
  - If you use the Z-Wave JS UI Docker 容器, you need at least 版本 `9.0.0`.
  - If you run your own Docker 容器 or some other 安装 method,
    you will need to 更新 your zwave-js-server instance to at least `1.32.0`.
