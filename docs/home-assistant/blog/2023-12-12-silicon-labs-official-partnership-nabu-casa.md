[Silicon Labs](https://www.silabs.com/) 已与 [Nabu Casa](https://www.nabucasa.com/) 建立官方合作关系，为我们的开源项目和硬件工作提供支持。

![Silicon Labs 与 Nabu Casa](/home-assistant/images/blog/2023-12-siliconlabs/siliconlabs-og.png)

Silicon Labs 是 Z-Wave 背后的公司，并为 Z-Wave、Zigbee、Thread 等多种标准设计芯片。他们的芯片为许多设备提供连接能力，包括 Philips Hue、Ring、IKEA TRADFRI，以及我们自己的 [Home Assistant Yellow](/home-assistant/yellow) 和 [Home Assistant SkyConnect](/home-assistant/skyconnect) 产品。事实上，迄今为止所有 Z-Wave 产品中的 Z-Wave 芯片都来自 Silicon Labs。

我们热爱开放标准，因为它们契合我们对智能家居的 [Open Home 理念](https://www.home-assistant.io/博客/2021/12/23/the-open-home/)：隐私、选择权与可持续性。这也是为什么 Nabu Casa 会利用 Home Assistant Cloud 订阅收入，大力投入这些开放标准的集成，而这意味着我们每天都会与 Silicon Labs 的技术打交道。比如，我们聘请了 [Dominic](https://github.com/alcalzone) 全职维护 [Z-Wave JS](https://github.com/zwave-js)，也聘请了 [Nikita](https://github.com/puddly) 全职维护 [Zigpy](https://github.com/zigpy/zigpy)——这是为 Home Assistant 提供 Zigbee 支持的底层库。还有其他开发者专注于确保我们自有硬件中的 Silicon Labs 芯片在 Home Assistant 中完美运行。

Z-Wave JS 是唯一的开源 Z-Wave 实现，如今已为 Home Assistant 之外越来越多的 Z-Wave 平台提供支持。我们的工作对 Z-Wave 生态的成长至关重要，而 Silicon Labs 通过这次合作对这一点给予认可，也让我们倍感欣喜。借助 Home Assistant，我们接触到大量运行 Z-Wave、Zigbee 和 Thread 的不同设备。如今作为合作伙伴，我们已经能够与 Silicon Labs 协作报告错误，并优先推动这些问题得到修复。
