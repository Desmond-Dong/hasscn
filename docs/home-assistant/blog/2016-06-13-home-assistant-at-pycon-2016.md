# Home Assistant at PyCon 2016

距离几位 Home Assistant 开发者前往波特兰参加 [PyCon 2016]（一场关于 Python 的大会）已经快两周了。我们此行既是为了学习各种让代码更好的技巧，更重要的是为了交流 Home 自动化。

<p class='img'>
  <img src='/home-assistant/images/blog/2016-06-pycon/crew.jpg' alt='Home Assistant 开发者' />
  几位 Home Assistant 开发者。从左到右分别是：[Paulus (@balloob)]、[Alex (@infamy)]、[Ryan (@rmkraus)]。
</p>

周一时，我（Paulus）面向 400 多位听众做了一场关于 Home Assistant 的演讲！一开始确实有点紧张，但几分钟后就顺利起来了，之后的提问环节也非常精彩。演示文稿可以在[这里][slides]查看，演讲视频就嵌在下方：

<div class='videoWrapper'>
<iframe width="560" height="315" src="https://www.youtube.com/embed/UhccJacWhdM" frameborder="0" allowfullscreen></iframe>
</div>

最让我印象深刻的一件事，是有很多人主动来告诉我们：他们有多喜欢 Home Assistant，它如何取代了他们之前的方案，他们多么享受为 Home Assistant 做贡献，以及我们的社区有多么乐于助人。这让我为 Home Assistant，尤其是我们的社区，感到无比自豪。

<!--more-->
PyCon 有几个我在其他大会上很少见到的优秀形式：开放交流（open spaces）和开发冲刺（sprints）。开放交流让任何人都有机会申请一个房间，组织一场一小时、主题不限的讨论。开发冲刺则在 PyCon 主会程结束后进行，连续四天都有房间供参与者一起协作，动手开发自己喜欢的开源项目。

我的演讲留给问答的时间有限，所以开放交流正好提供了一个很好的机会，既能回答所有未尽问题，也能和社区更深入地建立联系。第一天结束后大家反响很热烈，于是我们第二天又加开了一场开放交流。

我们的开放交流获得了非常积极的反馈，以至于 [Jonathan Baginski][@patchedsoul] 决定在线上再办一次。本月晚些时候我们将举办一场免费的线上研讨会 [Home Assistant Support 101 - Getting around in Home Assistant][webinar]，记得提前报名。

PyCon 会议部分结束后，我们又额外花了一天组织 Home Assistant 冲刺活动。这让我们可以帮助大家上手参与 Home Assistant 开发，也因此产出了一些非常棒的贡献。

我在 PyCon 度过了一段非常美好的时光。能和大家线下见面真的太棒了，希望明年还能见到你们中的很多人！
