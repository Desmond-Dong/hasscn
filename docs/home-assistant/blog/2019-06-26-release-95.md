## 房车助手

Home Assistant is not just limited to homes. Recently we had a boat owner on [Home Assistant podcast episode 48](https://hasspodcast.io/ha048/), and Home Assistant also powers recreational vehicles (RV) ([like this schoolbus](https://thisisthebus.com/our-tech-stack.html)).

With this 发布, we're making integrating Home Assistant into moving vehicles so much easier. In the last 发布, we introduced support for the 核心 config to be updated without restarting Home Assistant. This 发布 brings it a step further and introduces a new 服务 `homeassistant.set_location`. With this 服务, you can automatically change the location that Home Assistant is located. Once updated, it automatically updates all the things that rely on the location, like your sunset 触发器 and your current 天气 (only [Met][met docs] supported now).

## AdGuard Home - 本地广告与追踪器拦截

Here at Home Assistant, we're big 风扇 of privacy and we're always working on improving it. So this brings us to [AdGuard](https://adguard.com/). AdGuard Home is a local DNS server that automatically blocks any 设备 looking up a domain that is used to serve ads, spreads malware, or isn't appropriate for your kids. Thanks to [@frenck], starting 发布 0.95, it has a top-notch 集成 with Home Assistant:

 - One-click 安装 the AdGuard 插件 from the [Community Hass.io Add-ons](https://github.com/hassio-addons/repository/blob/master/README.md).
 - Home Assistant automatically detects the Hass.io 插件 and offers to set it up. (It works with standalone installs too, see the [AdGuard 集成 文档][adguard docs] on how to 配置 it.)
 - DONE!

One cool new feature: because parental control is now available as a 开关, you can control AdGuard via Alexa and Google: "Hey Google, 打开 parental control" 😲.

## Life360

More goodies. Keep paying attention as we're not done yet. Also new in 0.95 is the [Life360 集成][life360 docs] thanks to the hard work by [@pnbruckner]. 此integrations started its life a little over 360 days ago [on the forums as a custom 集成](https://community.home-assistant.io/t/life360-devices-tracker-platform/52406) and has now become part of Home Assistant. Life360 is an app to allow friends or family members to share location with each other.

If you are migrating from the custom 集成, [please read the instructions](https://community.home-assistant.io/t/life360-devices-tracker-platform/52406/1077).

## 改进后的 Alexa 集成

One more thing… This 发布 improves the Home Assistant Cloud Alexa 集成. Just like Google Assistant, it is now possible to manage which 实体 are exposed to Amazon via the 用户 interface. Alexa actually offers some cool features around this. Any change you make is instantly reflected inside Alexa. Add an 实体? Control it instantly via Alexa.

However, we didn't stop there. The [manual installations of Alexa](/home-assistant/integrations/alexa.smart_home/#alexa-component-configuration) has had optional proactive 状态 reporting since earlier this year, and this is now available for Home Assistant Cloud 用户 too. If you choose to opt-in to this, every 状态 change of your exposed 实体 is sent to Amazon as soon as they happen. It allows Amazon Echos with displays, to instantly reflect the 状态 and allows you to create routines inside Amazon powered by your Home Assistant 实体.

## 值得注意的不兼容变更

For the last couple of releases, we've been printing 警告 if your 配置 contained invalid slugs or extra keys. Starting with this 发布, these 警告 are now 错误. No procrastination anymore, you'll need to fix it now.

We have also realized that the default database clean up task never automatically ran as intended. Instead of running every night, cleaning up your history so that it only contains the last 10 days, it never ran. Oopsie. We've fixed it in this 发布, so now the defaults kick in. Want to adjust how much history you want to retain? Check [the docs][recorder docs].
