Home Assistant 是一个强大的工具，可以通过自动化家居为你节省时间和金钱。家中很多重要的事项，从供暖到自动喂鱼器，都可能依赖它运行。不过，随着你的设置越来越复杂，维护它——我们称之为 *“数字家务”*——也会变得越来越耗时。备份是这项维护中至关重要的一部分，但它往往会被忽视，尤其是新用户，常常要等到灾难真正发生时才意识到它的重要性。

为了让备份变得毫不费力，我们在 [Home Assistant 最新版本](/home-assistant/blog/2025/01/03/release-20251/)中推出了全新的自动备份和异地备份功能。这些改进让你以前所未有的方式保护自己的设置。新年伊始，我们也想鼓励你立下一个简单却关键的目标：定期备份你的数据。借助这些新工具，只需五分钟就能完成设置，并带来长久的安心。

继续阅读下方内容了解更多，或者现在就通过[更新](https://my.home-assistant.io/redirect/updates/)并访问[备份](https://my.home-assistant.io/redirect/备份/)页面开始设置。

## 备份里有什么？

备份是打造可靠智能家居的基础。它可以在硬件故障、自然灾害和网络攻击发生时保护你的系统，让你能够快速恢复运行。一套可靠的备份策略遵循 3-2-1 原则：

- 3 份数据副本
- 2 份存储在不同介质上
- 1 份保存在异地

这听起来也许有些夸张，但想象一下遭遇雷击的情况：你的 Home Assistant 系统和本地存储，比如网络附加存储（NAS），都有可能同时损坏。这就是为什么异地备份是实现真正韧性的必要条件。一套完整的备份策略应当包括：

- 定期且频繁地备份关键数据
- 遵循 3-2-1 冗余原则
- 对备份进行加密，防止被窃取或损坏
- 测试备份，确保它们可以正常使用
- 明确备份的保留时长

为支持这些最佳实践，我们投入了大量工作，推出了自动备份功能，并与异地备份位置提供商实现了无缝集成，比如 [Home Assistant Cloud](/home-assistant/cloud/)。这些更新是 [我们 2024 路线图](/home-assistant/blog/2024/06/12/roadmap-2024h1/#home-assistant-cloud-peace-of-mind) 的一部分，它们让用户更容易建立可靠的备份策略，并让你的智能家居持续稳定运行。

## 备份从未如此简单

<img src='/home-assistant/images/blog/2025-01-backups/backups.png' style='border: 0;box-shadow: none;' alt="备份页面">

Home Assistant 的备份能力一直都非常出色，它会备份每一项设置和附加组件，让恢复过程几乎像魔法一样。它甚至支持跨不同 CPU 架构恢复！不过直到最近，在 Home Assistant 中配置一套 3-2-1 备份策略还不算真正简单。随着 Home Assistant 2025.1 的到来，这一点改变了，我们通过三个重要新功能大幅提升了备份体验。

- **自动加密备份** - 你现在可以安排每晚或每周自动执行备份，确保始终手边有最近的备份可用。
- **自动清理** - 你不再需要担心备份占满磁盘空间，因为现在可以设置最多保留多少份备份。
- **备份位置** - 我们新增了一套系统，让集成可以提供备份存储位置（也称为 [备份代理](https://开发者.home-assistant.io/docs/core/platform/备份)），让你能够将备份同步到本地或异地存储。首发支持 [网络共享](https://my.home-assistant.io/redirect/storage/) 和 [Home Assistant Cloud](https://my.home-assistant.io/redirect/cloud/)（订阅用户无需额外付费即可使用），未来还会有更多位置加入。

### 使用我们的备份向导开始设置

<img src='/home-assistant/images/blog/2025-01-backups/wizard.png' style='border: 0;box-shadow: none;' alt="备份向导">

Home Assistant 2025.1 中的新自动备份功能附带了一个全新且易用的备份向导。只需点几下，你就可以按固定计划定期备份 Home Assistant 系统，并轻松选择备份的存储位置。新用户会看到这个向导，以确保他们完成自动加密备份设置。选择推荐设置后，系统会每晚创建备份、保留三份副本，并上传到所有可用位置。

### 备份位置

<img src='/home-assistant/images/blog/2025-01-backups/locations.png' style='border: 0;box-shadow: none;' alt="备份位置">

在首发阶段，可用的备份位置还不算多，但我们已经让开发者可以非常轻松地集成更多位置，因此未来会有大量选择陆续加入。作为我们在 [路线图中](/home-assistant/blog/2024/06/12/roadmap-2024h1/#next-priority-encrypted-cloud-备份) 提出的智能家居韧性重点工作之一，[Home Assistant Cloud](/home-assistant/cloud/) 现已支持云备份，所有订阅用户从第一天起都可免费使用。它只会存储最新的一份备份，且该备份必须小于 5GB（如果文件太大无法上传，你会收到修复通知）。如果你丢失了其他备份，始终可以前往[你的 Cloud 账户页面](https://account.nabucasa.com/)下载最新备份。

像这样强大的 Home Assistant 新功能，离不开 Home Assistant Cloud 订阅用户的支持。将备份功能免费加入订阅服务，是我们对他们持续支持表达感谢的最起码方式。

### 默认安全

<img src='/home-assistant/images/blog/2025-01-backups/encrypt.png' style='border: 0;box-shadow: none;' alt="加密密钥页面">

备份中包含所有 Home Assistant 数据，包括你智能家居中每个已集成设备的密钥，以及你生活方式的历史记录。这些数据非常重要，必须妥善保护。随着自动备份和我们鼓励用户采用的 3-2-1 策略越来越普及，备份会被存储到异地，因此所有自动备份都将进行完整加密。

当你配置自动备份时，系统会向你展示加密密钥，并提供下载紧急工具包的选项。紧急工具包包含所有重要信息，其中也包括一条指向[如何使用它来恢复备份的说明](/home-assistant/more-info/backup-emergency-kit)的链接。如果你之后不小心删除了紧急工具包，也可以在[备份配置页面](https://my.home-assistant.io/redirect/备份/)重新下载。
