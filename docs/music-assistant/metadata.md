# 元数据提供者 - 概述

Music Assistant 中可用的元数据提供者用于补充从音乐提供者获取的项目中缺失的元数据。它们不会更改从原始项目获取的任何内容（例如 ID3 标签），因此通常不应禁用它们。

## 歌词

歌词可以从各种来源获取，具体取决于音乐提供者。

对于<a href="/music-assistant/music-providers/filesystem/#known-issues-notes">本地文件提供者</a>，歌词可以嵌入在标签中或 `.lrc` 文件中。这是显示歌词最可靠的方式。

对于所有提供者，如果一个或多个歌词元数据提供者可用，则会尝试根据以下规则获取歌词（注意可能无法找到匹配的歌词）

- 当首次播放曲目时，将排队一个任务来获取歌词，这意味着下次播放该曲目时歌词应该可用（这些任务需要一些时间才能完成）
- 如果在上次更新后至少 90 天播放曲目，则会发生元数据更新
- 如果查看特定曲目并从右上角菜单中选择更新元数据，这将触发歌词更新请求

Tidal 具有原生歌词支持。由于复杂的 API 查找过程，歌词仅在完整曲目查找期间获取，这仅在查看曲目信息或播放曲目时发生。此时您还必须刷新项目，然后可能仍需要再次点击播放才能显示歌词。

> [!NOTE]
> lrclib 提供者需要匹配的曲目名称、艺术家、专辑和时长（2 秒内）才能返回结果
