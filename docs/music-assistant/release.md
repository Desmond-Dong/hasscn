# 发布周期 <img src="/assets/icons/release-cycle-icon.png" alt="预览图片" style="width: 70px; float: right;"  loading="lazy" />

## 稳定版

稳定版只会每 1-3 个月发布一次，具体取决于功能改进的数量。这些稳定版将在测试周期结束后发布，并发现足够稳定以进行推广。

如果有迫切需要（例如紧急错误修复），可能会推送补丁版本

## 测试版

除了稳定版通道外，还有测试版通道，有专用的 Home Assistant 应用（和 Docker 镜像）。
一旦发布新的稳定版本，将开始新的测试周期（次版本号增加）。
例如，一旦发布稳定版 2.3，2.4 版的测试版就开始了。

测试版计划每 1-4 周发布一次，具体取决于更改数量。
在某些情况下，例如正在开发大型功能时，预计每周会有多个测试版发布。

### 每夜版

每天晚上，如果至少合并了两个 PR，将构建自动每夜版。如果您想跟上 Music Assistant 最新的前沿开发，请安装此版本！使用风险自负，可能不建议用于日常/生产使用，因为它可能不稳定。

### 开发通道

这是 Music Assistant 的特殊开发应用，允许开发人员直接在 Home Assistant 中快速测试特定分支、拉取请求，甚至 Music Assistant 的分支。

## 并行运行服务器版本

可以并行运行稳定版、测试版、每夜版或开发版服务器应用，因为它们不共享任何数据。因此，例如，您可以临时运行测试版应用来试用新功能，然后恢复到稳定版本。您可以通过手动停止和启动相关服务器来完成此操作。不要在同一个主机上同时运行两个服务器。

> [!CAUTION]
> 请注意不要意外删除应用，因为这将导致所有数据丢失（如果您没有备份）。这也适用于 Docker 用户：请注意数据文件夹，并注意数据文件夹不向后兼容。不要尝试在通道之间使用它！
